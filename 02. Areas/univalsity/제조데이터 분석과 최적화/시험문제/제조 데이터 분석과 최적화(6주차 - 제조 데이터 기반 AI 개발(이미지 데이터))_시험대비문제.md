## 📌 핵심 내용 정리

> [!note] 학습 포인트
> 아래 내용을 먼저 읽고, 문제를 풀어보세요. 이 강의는 크게 두 축으로 구성됩니다: (1) 크로메이트 데이터셋을 활용한 지도 학습 기반 CNN 분류 모델, (2) MVTec AD 데이터셋을 활용한 비지도 학습 기반 합성곱 오토인코더(CAE) 이상 탐지 모델.

### 1. 크로메이트(Chromate) 공정과 데이터셋

크로메이트는 중크롬산염을 이용해 방청 피막을 형성하는 표면처리 공정으로, 광택이 필요한 하이엔드 제품에 적용된다. 주반응은 액 중의 산이 아연을 용해하고, 중크롬산 이온이 환원되면서 아연 표면에 복잡한 성분의 피막이 침착된다. 이 강의에서는 품질검사 자동화를 목표로 이미지 분류 AI를 개발한다.

- 전체 이미지: 1,452개 (정상 1,378개 / 불량 74개) — **심각한 클래스 불균형** 존재

### 2. CNN이 필요한 이유 — MLP의 한계

이미지 데이터는 세 가지 특성을 갖는다: **지역 상관성(Local Correlation)**, **지역적 균질성**, **구성적/계층적 구조**. 기존 MLP는 두 가지 치명적 한계를 갖는다.

- **파라미터 폭발**: 224×224×3 이미지를 100개 뉴런 MLP에 입력하면 파라미터 수가 약 1,500만 개(15,052,800)에 달해 일반화 성능이 저하되고 메모리가 낭비된다.
- **공간 구조 무시**: MLP는 이미지를 1차원으로 Flatten하므로 픽셀의 이웃성과 위치 구조를 전혀 활용하지 못한다.

CNN(Convolutional Neural Network)은 컨볼루션 연산으로 지역 특징을 추출하여 이 두 한계를 동시에 극복한다.

### 3. 컨볼루션(Convolution) 연산

컨볼루션은 필터(커널)를 이미지 위에 슬라이딩하며 원소별 곱의 합을 계산하는 연산이다. 6×6 입력에 3×3 필터를 적용하면 4×4 특징 맵(Feature Map)이 생성된다. 과거 Computer Vision에서는 엣지 검출용 Sobel 필터 등을 수동으로 설계했으나, CNN은 이 필터 값(가중치)을 학습으로 자동 결정한다.

### 4. 데이터 파이프라인 (step1_data_analysis.py)

| 단계 | 도구 | 역할 |
|------|------|------|
| 환경 설정 | Setup | 라이브러리 및 경로 설정 |
| 전처리 | transforms.Compose | Resize(256,256) + ToTensor() 순서로 변환 파이프라인 구성. ToTensor()는 0-255 픽셀값을 0-1로 정규화 |
| 데이터셋 | ImageFolder | 폴더 이름을 자동으로 클래스 레이블로 인식 (정상→0, 불량→1) |
| 미니배치 | DataLoader | batch_size 단위로 데이터를 잘라 제공. shuffle=True로 순서 무작위화 |
| 시각화 | Matplotlib | PyTorch 텐서(C,H,W)를 permute(1,2,0)으로 (H,W,C)로 변환 후 표시 |

**PyTorch vs Matplotlib 텐서 축 순서**:
- PyTorch: **(Channel, Height, Width)** — AI 모델 연산에 최적화
- Matplotlib: **(Height, Width, Channel)** — 화면 표시에 최적화
- 변환: `image.permute(1, 2, 0).numpy()`

### 5. SimpleCNN 모델 설계 (step2_train.py)

훈련 파이프라인: 모델 구조 설계(CNN) → 디바이스 설정(GPU/CPU) → 학습 도구 설정(Loss & Optimizer) → 학습 루프(Training) → 모델 저장(Save)

**특징 추출기(Feature Extractor)**:
- `nn.Conv2d(3, 32, kernel_size=3, padding=1)` → `nn.ReLU()` → `nn.MaxPool2d(kernel_size=2, stride=2)`
- `nn.Conv2d(32, 64, kernel_size=3, padding=1)` → `nn.ReLU()` → `nn.MaxPool2d(kernel_size=2, stride=2)`

**분류기(Classifier)**:
- `nn.Flatten()` → `nn.Linear(64×64×64, 128)` → `nn.ReLU()` → `nn.Dropout(0.5)` → `nn.Linear(128, 2)`

핵심 컴포넌트:
- **nn.Module**: 모든 PyTorch 신경망 모델이 상속해야 하는 기반 클래스
- **nn.Conv2d**: 이미지에서 선, 색상, 질감 등 특징(Feature) 추출
- **nn.MaxPool2d**: 이미지 크기를 절반으로 줄이고 가장 중요한 특징만 남겨 연산량 감소 및 과적합 방지
- **nn.Dropout(0.5)**: 학습 중 무작위로 50% 뉴런을 끄는 정규화 기법으로 과적합(Overfitting) 방지
- **nn.Linear**: 완전 연결 층으로 최종 클래스(정상/불량) 결정
- **forward 함수**: 데이터가 모델에 입력될 때 층을 통과하는 순서 정의

### 6. 모델 평가 (step3_evaluate.py)

**평가 모드 설정**:
- `model.eval()`: Dropout 등 훈련용 레이어를 비활성화해 실전 성능 발휘
- `with torch.no_grad()`: 기울기 계산을 중단하여 메모리 절약 및 평가 속도 극대화

**추론 파이프라인**: Raw Output(Logit) → Softmax(확률 변환, 0~1) → Argmax(최종 예측 클래스 선택) → Extend(결과 저장)

**성능 지표**:
- **Accuracy(정확도)**: 전체 문제 중 맞힌 비율
- **Precision(정밀도)**: 모델이 '불량'으로 예측한 것 중 진짜 불량의 비율 (오탐지 방지)
- **Recall(재현율)**: 실제 '불량'인 것 중 모델이 놓치지 않고 찾아낸 비율 (미탐지 방지)
- **F1-Score**: Precision과 Recall의 조화 평균 (데이터 불균형 시 핵심 지표)
- **Confusion Matrix**: 모델 예측값과 실제 정답을 2×2 표로 나타낸 것
- **ROC Curve**: 판단 기준(임계값) 변화에 따른 모델 성능 시각화
- **AUC(Area Under the Curve)**: ROC Curve 아래 면적 (1에 가까울수록 우수)

### 7. 추론 및 설명 가능 AI (step4_inference.py)

**Grad-CAM(Gradient-weighted Class Activation Mapping)**: AI가 이미지의 어느 부분을 보고 판단을 내렸는지 시각화하는 XAI 기법.

**Hook 메커니즘**: 모델 내부 구조를 수정하지 않고 특정 레이어에 도청 장치를 달아 데이터 통과 시 정보를 복사·저장하는 방식
- **forward_hook**: 이미지가 층을 통과할 때 생성되는 특징 맵 저장
- **backward_hook**: 역전파 시 발생하는 기울기 저장

Grad-CAM은 **마지막 합성곱 층**을 대상으로 사용한다 — 이 층이 이미지 전체 맥락을 가장 잘 이해하기 때문이다. OpenCV의 JET Colormap을 적용하여 열화상 카메라처럼 시각화한다.

### 8. MVTec AD와 합성곱 오토인코더(CAE)

**MVTec AD**: 독일 MVTec Software GmbH에서 발표한 산업용 이미지 이상 탐지 전용 데이터셋. 비지도 학습 기반 알고리즘 성능 평가에 최적화.
- 객체(10종): 병, 케이블, 캡슐, 헤이즐넛, 금속 너트, 알약, 나사, 칫솔, 트랜지스터, 지퍼
- 질감(5종): 카펫, 격자, 가죽, 타일, 나무

**비지도 학습 구조의 특징**:
- Train Set: 결함이 없는 정상 이미지만으로 구성
- Test Set: 정상 이미지 + 다양한 불량 이미지
- 픽셀 단위 정답지(Ground Truth Mask) 제공 → Segmentation 성능 평가 가능
- 실제 결함 유형: 스크래치, 오염, 짜그러짐, 부러짐 등

**합성곱 오토인코더(CAE) 구조**:
- **인코더(Encoder)**: Conv + Pooling을 반복하며 공간 차원을 줄이고 채널을 늘려 특징 추출
- **잠재 공간(Latent Space / Bottleneck)**: 고차원 이미지를 저차원으로 압축한 핵심 요약본. 잠재 벡터는 모델의 '핵심 요약본' 역할
- **디코더(Decoder)**: 잠재 벡터를 업샘플링(Upsampling) 또는 전치 합성곱(Transposed Convolution, Deconvolution)으로 원본 크기로 복원

**이상 탐지 원리**: 모델은 정상 이미지만 학습하므로 정상 이미지는 낮은 복원 오차(MSE Loss)를 보인다. 불량 이미지를 입력하면 모델이 학습하지 못한 결함 패턴이 복원되지 않아 높은 복원 오차가 발생 → **복원 실패 지점 = 결함**

**이상 점수(Anomaly Score) 산출**: `np.max(error_map)` — 픽셀 오차의 평균이 아닌 **최댓값**을 사용하는 이유는 제조 현장에서 미세한 스크래치 하나도 불량이며, 평균을 사용하면 넓은 정상 면적에 미세 불량 신호가 희석되기 때문이다.

**최적 임계값 결정**: `best_threshold = thresholds[np.argmax(f1_scores)]` — F1-Score가 최대인 지점을 최적 임계값으로 설정.

> ② 모델이 정상 패턴만 학습하므로 학습 시 본 적 없는 불량 패턴의 복원에 실패하여 높은 오차가 발생한다
> ③ 불량 이미지는 정상 이미지보다 픽셀 수가 많으므로 복원 오차가 자연히 커진다
> ④ Bottleneck의 크기를 불량 이미지 수에 맞게 조절하여 이상 패턴을 분리한다

> [!info]- 💡 정답 및 해설 보기
> **정답**: ②
>
> **해설**: CAE의 이상 탐지 원리는 '정상 패턴의 완벽한 학습 → 이상 패턴 복원 실패'이다. Train Set을 정상 이미지만으로 구성하여 모델이 정상 표면의 특징을 완벽히 학습한다. 추론 시 정상 이미지는 낮은 MSE Loss로 복원되지만, 불량 이미지의 결함 영역은 모델이 학습하지 못한 패턴이므로 복원에 실패하여 높은 MSE Loss가 발생한다. 이 복원 실패 지점이 바로 결함이다.

> ② 모델 내부 구조(코드)를 수정하지 않고 특정 층의 특징 맵과 기울기를 비침습적으로 수집하기 위해
> ③ 모델 추론 시 메모리 사용량을 줄이기 위해
> ④ 여러 층의 특징 맵을 동시에 합산하여 더 정확한 활성화 맵을 생성하기 위해

> [!info]- 💡 정답 및 해설 보기
> **정답**: ②
>
> **해설**: Hook(훅)의 핵심 가치는 **비침습성**이다. 모델의 `forward()` 함수나 내부 레이어 코드를 수정하면 모델 자체가 변경되어 원본 동작에 영향을 줄 수 있다. Hook은 외부에서 특정 레이어에 "도청 장치"처럼 부착하여 데이터가 통과할 때 정보를 몰래 복사·저장하므로, 모델 내부를 전혀 수정하지 않고도 필요한 정보(특징 맵, 기울기)를 수집할 수 있다.

> [!example]- 💡 모범 답안 및 채점 포인트
> **모범 답안**: MLP의 첫 번째 한계는 파라미터 폭발이다. 이미지를 1차원으로 Flatten하여 완전 연결하면 파라미터 수가 이미지 크기에 비례하여 폭발적으로 증가한다(224×224×3 이미지를 100개 뉴런에 연결하면 약 1,500만 개). 이는 일반화 성능 저하와 메모리 낭비를 초래한다. CNN은 작은 필터(커널)를 이미지 전체에 공유하는 파라미터 공유(Parameter Sharing)를 통해 파라미터 수를 극적으로 줄인다. 두 번째 한계는 공간 구조 파괴다. MLP는 이미지를 Flatten하므로 픽셀의 위치 정보와 이웃 관계가 사라진다. CNN은 2차원 공간 구조를 유지한 채 지역적 컨볼루션 연산을 수행하므로 인접 픽셀 간의 공간 상관성(Local Correlation)을 효과적으로 활용한다.
>
> **채점 포인트**:
> - [ ] MLP의 파라미터 폭발 문제와 구체적 예시(이미지 크기 → 파라미터 수) 언급
> - [ ] MLP의 공간 구조 무시 문제(Flatten으로 인한 위치 정보 소실) 언급
> - [ ] CNN의 파라미터 공유로 파라미터 수 감소 설명
> - [ ] CNN의 2차원 공간 구조 보존을 통한 지역 상관성 활용 설명

### 문제 2 `[기본]`

**`model.eval()`과 `with torch.no_grad():`의 역할을 각각 구분하여 설명하고, 평가 시 두 가지를 모두 사용해야 하는 이유를 서술하시오.**

> [!example]- 💡 모범 답안 및 채점 포인트
> **모범 답안**: `model.eval()`은 모델을 평가 모드로 전환하는 명령으로, Dropout과 BatchNorm처럼 훈련과 평가 시 다르게 동작하는 레이어의 동작 방식을 변경한다. Dropout의 경우 훈련 중에는 무작위로 뉴런을 끄지만, eval() 후에는 모든 뉴런을 활성화하여 100% 실력을 발휘한다. `with torch.no_grad():`는 이 컨텍스트 내에서 기울기(Gradient) 계산을 중단하는 명령이다. 평가 시에는 가중치를 업데이트할 필요가 없으므로 역전파를 위한 기울기 계산이 불필요하며, 이를 비활성화함으로써 메모리 사용량을 줄이고 추론 속도를 크게 향상시킨다. 두 명령은 완전히 독립적인 역할을 하므로 평가 시에는 반드시 함께 사용해야 정확한 평가가 가능하다.
>
> **채점 포인트**:
> - [ ] `model.eval()`이 Dropout/BatchNorm 등 레이어 모드를 변환한다고 설명
> - [ ] `torch.no_grad()`가 기울기 계산을 중단하여 메모리 절약 및 속도 향상을 한다고 설명
> - [ ] 두 명령이 독립적이며 기능이 중복되지 않음을 명시
> - [ ] 평가 시 두 가지를 모두 사용해야 하는 이유 논리적으로 서술

**크로메이트 품질 검사와 MVTec AD 이상 탐지는 모두 제조 이미지 데이터를 활용한 AI이지만, 접근 방식이 근본적으로 다르다. 두 접근법(지도 학습 기반 CNN 분류 vs. 비지도 학습 기반 CAE 이상 탐지)의 데이터 구조, 모델 아키텍처, 이상 탐지 원리를 비교하고, 각 방식이 실제 제조 현장에서 적합한 상황을 논하시오.**

> [!example]- 💡 모범 답안 및 채점 포인트
> **모범 답안**:
>
> 지도 학습 기반 CNN 분류(크로메이트)는 정상과 불량 이미지가 모두 레이블링된 상태로 학습에 사용된다. SimpleCNN은 합성곱-풀링 블록으로 특징을 추출하고, 완전 연결 층으로 최종 클래스를 결정하는 분류 모델이다. 모델은 훈련 과정에서 정상과 불량의 시각적 차이를 직접 학습하며, 추론 시 Softmax로 각 클래스의 확률을 출력한다. 이 방식은 불량 샘플을 충분히 확보할 수 있는 경우에 적합하나, 크로메이트 데이터셋처럼 불량 샘플이 극히 적은(74개) 클래스 불균형 상황에서는 성능 저하와 과적합 위험이 존재한다.
>
> 비지도 학습 기반 CAE(MVTec AD)는 정상 이미지만으로 학습하고, 복원 오차를 이상 점수로 활용한다. CAE는 인코더-잠재 공간-디코더 구조로 이미지를 압축 후 복원하며, 정상 패턴만 완벽히 학습한 모델은 불량 이미지 복원에 실패하여 높은 MSE Loss를 보인다. 이 방식은 불량 샘플이 거의 없거나 전혀 없어도 적용 가능하고, 픽셀 단위 결함 위치까지 시각화할 수 있다는 장점이 있다.
>
> 결론적으로, 불량 데이터를 충분히 수집할 수 있고 결함 유형이 명확히 정의된 환경에서는 CNN 분류가, 불량 데이터가 희소하거나 미지의 결함 유형까지 탐지해야 하는 환경에서는 CAE 기반 이상 탐지가 더 적합하다.
>
> **핵심 채점 포인트**:
> - [ ] 지도 학습(CNN): 정상+불량 레이블 데이터 필요, 클래스 분류 모델임을 명시
> - [ ] 비지도 학습(CAE): 정상 데이터만으로 학습, 복원 오차로 이상 탐지함을 명시
> - [ ] 두 모델의 아키텍처 차이 비교(CNN 분류기 vs. 인코더-Bottleneck-디코더)
> - [ ] 각 방식이 적합한 실제 상황 논리적으로 제시 (불량 데이터 수량 기준)
> - [ ] 클래스 불균형 문제와의 연결 (가점 항목)

### 문제 2 `[심화]`

**합성곱 오토인코더(CAE)에서 잠재 공간(Latent Space)이 "복사기"가 아닌 "모래시계"로 비유되는 이유를 설명하고, Bottleneck의 크기(잠재 벡터 차원)가 이상 탐지 성능에 미치는 영향을 과소 표현과 과다 표현의 두 관점에서 분석하시오.**

> [!example]- 💡 모범 답안 및 채점 포인트
> **모범 답안**:
>
> CAE의 목적은 단순한 이미지 복사가 아니라 정보의 취사선택이다. 복사기는 모든 정보를 그대로 재현하지만, 모래시계는 넓은 입력을 좁은 통로(Bottleneck)로 강제 통과시켜 핵심 정보만 남기고 불필요한 노이즈를 제거한다. CAE가 정상 이미지로만 학습할 때 Bottleneck은 정상 표면의 핵심 패턴(질감, 색상, 규칙적 패턴 등)만 담도록 최적화된다. 결함처럼 낯선 패턴은 이 좁은 통로를 제대로 통과하지 못하고, 디코더가 학습한 정상 패턴으로 복원을 시도하다 실패하여 높은 복원 오차가 발생한다.
>
> Bottleneck 크기가 너무 작으면(과소 표현), 정상 이미지의 필수 정보도 충분히 담지 못하여 정상 이미지조차 복원하지 못하고 높은 오차를 보인다. 이 경우 모든 이미지가 '이상'으로 판정되는 문제가 발생한다. 반대로 Bottleneck 크기가 너무 크면(과다 표현), 모델이 불량 패턴까지 포함한 모든 정보를 담을 수 있어 불량 이미지도 높은 품질로 복원된다. 이 경우 정상과 불량 간 복원 오차 차이가 줄어들어 이상 탐지 성능이 저하된다. 따라서 Bottleneck 크기는 정상 패턴은 충분히 표현하되 이상 패턴은 표현하기 어려운 최적의 균형점을 찾아야 한다.
>
> **핵심 채점 포인트**:
> - [ ] "모래시계" 비유 — 정보 취사선택(핵심 보존, 노이즈 제거) 설명
> - [ ] Bottleneck이 정상 패턴만 학습 → 이상 패턴 복원 실패 메커니즘 서술
> - [ ] 과소 표현(Bottleneck 너무 작음): 정상 복원 실패, 모든 이미지 이상 판정
> - [ ] 과다 표현(Bottleneck 너무 큼): 불량도 완벽 복원, 탐지 성능 저하
> - [ ] 최적 Bottleneck 크기 결정의 트레이드오프 언급 (가점 항목)

### 문제 3 `[심화]`

**제조 데이터 분석에서 6×6 입력 이미지에 3×3 Vertical Edge Filter(필터값: 상단 행 [1,0,-1], 중단 행 [1,0,-1], 하단 행 [1,0,-1])를 Stride=1, Padding=0으로 적용할 때 출력 특징 맵의 크기를 계산하고, 이 필터가 수직 엣지를 검출하는 원리를 수식과 연결하여 설명하시오.**

> [!example]- 💡 모범 답안 및 채점 포인트
> **모범 답안**:
>
> 출력 특징 맵 크기 공식: Output = (Input - Filter + 2×Padding) / Stride + 1 = (6 - 3 + 2×0) / 1 + 1 = 4. 따라서 출력은 **4×4** 특징 맵이다.
>
> Vertical Edge Filter의 동작 원리: 이 필터의 첫 번째 열은 모두 +1, 두 번째 열은 0, 세 번째 열은 모두 -1이다. 필터가 이미지의 특정 위치에 적용될 때, 왼쪽 3열의 픽셀값 합에서 오른쪽 3열의 픽셀값 합을 뺀 값이 출력된다. 이미지 내에 픽셀값이 급격히 변하는 수직 경계(밝은 영역에서 어두운 영역으로의 전환)가 있으면, 왼쪽과 오른쪽의 픽셀값 차이가 크게 나타나 출력값의 절댓값이 커진다(강한 엣지 반응). 반면 픽셀값이 균일한 영역에서는 좌우 차이가 없어 출력이 0에 가까워진다(엣지 없음). 강의 예시에서 3×1+1×1+2×1+0×0+5×0+7×0+1×(-1)+8×(-1)+2×(-1) = 6 - 11 = -5로 계산된다.
>
> **핵심 채점 포인트**:
> - [ ] 출력 크기 공식 `(Input - Filter + 2P) / S + 1`을 정확히 적용하여 4×4 도출
> - [ ] 필터의 +1/-1 구조가 좌우 픽셀값 차이를 계산함을 설명
> - [ ] 수직 경계에서 좌우 픽셀값 차이가 커져 출력값 절댓값이 증가함을 설명
> - [ ] 강의 예시(결과 -5) 또는 유사 계산으로 구체적 수식 제시 (가점 항목)

### 문제 4 `[심화]`

**이 강의에서 크로메이트 데이터셋의 클래스 불균형(정상 1,378개 : 불량 74개) 문제가 모델 성능 평가와 모델 훈련에 각각 어떤 위협을 초래하는지 분석하고, 이를 완화하기 위한 방법론을 데이터 수준과 알고리즘 수준에서 각각 제안하시오.**

> [!example]- 💡 모범 답안 및 채점 포인트
> **모범 답안**:
>
> 성능 평가 측면에서, 정상:불량 비율이 약 18.6:1인 경우 모든 샘플을 정상으로 예측해도 Accuracy가 약 94.9%(=1378/1452)에 달한다. 이는 불량 탐지 능력이 전혀 없는 모델도 높은 Accuracy를 기록하게 만들어, Accuracy 지표가 무의미해진다. 이 강의에서 Precision, Recall, F1-Score, Confusion Matrix, AUC를 함께 사용하는 이유가 바로 이 때문이다.
>
> 모델 훈련 측면에서, 손실 함수(Loss)를 최소화하는 학습 과정에서 모델이 다수 클래스(정상)에 편향되어 소수 클래스(불량)를 무시하는 방향으로 학습될 위험이 있다. 결국 불량을 정상으로 분류하는 미탐지(False Negative)가 증가한다.
>
> 데이터 수준 완화 방법: 불량 이미지를 인위적으로 늘리는 데이터 증강(Data Augmentation — 회전, 반전, 밝기 조절, Mixup 등)을 적용하거나, 정상 이미지를 줄이는 언더샘플링(Undersampling) 또는 SMOTE 같은 합성 오버샘플링을 사용할 수 있다.
>
> 알고리즘 수준 완화 방법: 손실 함수에 클래스 가중치(Class Weight)를 적용하여 소수 클래스(불량)의 오분류에 더 큰 페널티를 부여하거나, Focal Loss처럼 어려운 샘플에 집중하는 손실 함수를 사용할 수 있다.
>
> **핵심 채점 포인트**:
> - [ ] 평가 위협: Accuracy 지표의 무의미화 메커니즘과 수치적 근거 제시
> - [ ] 훈련 위협: 다수 클래스 편향 학습 및 False Negative 증가 설명
> - [ ] 데이터 수준 해결책(증강, 오버샘플링 등) 구체적 방법 제시
> - [ ] 알고리즘 수준 해결책(클래스 가중치, Focal Loss 등) 구체적 방법 제시 (가점 항목)
