## 📌 핵심 내용 정리

> [!note] 학습 포인트
> 아래 내용을 먼저 읽고, 문제를 풀어보세요. 이 강의는 두 가지 실제 산업 사례(용접 예지보전, AI4I 2020)를 통해 수치 데이터 기반 AI 개발의 전 과정을 다룹니다.

### I. 용접 예지보전 (Welding Predictive Maintenance)

**분석 목적**: 용접 설비(용접기)에서 발생하는 이상을 사전에 식별하여 설비 고장을 예방하는 예지보전 시스템 구축. 전류, 전압, 용접 속도 등 실시간 센서 데이터를 활용하여 이상 데이터를 사전에 탐지한다.

**데이터셋 구성**:
- 수집 변수: DV_R(직류 전압), DA_R(직류 전류), AV_R(교류 전압), AA_R(교류 전류), PM_R(용접 속도)
- 수집 장비: 용접 설비 내 센서 및 PLC, 수집 주기 5 min/pipe, CSV 파일로 저장
- 정상 데이터: 654,200개 (81.7%), 이상 데이터: 145,800개 (18.3%) — 총 800,000 Rows

**AI 개발 파이프라인**: 설비(Welding Machine) → 센서 & PLC(실시간 공정 변수 감지) → 데이터 수집(CSV) → AI 알고리즘(예지보전 모델 입력 데이터화)

**모델 선정 — LSTM-Autoencoder 채택 이유**:

| 비교 기준 | 전통적 기계학습(지도학습) | LSTM-Autoencoder(비지도학습) |
|-----------|--------------------------|------------------------------|
| 요구 데이터 | 정상·고장 데이터의 균형적 확보 필수 | 정상 데이터만으로 학습 가능 (데이터 불균형 해결) |
| 시간적 흐름 | 단일 시점 데이터를 독립적으로 판단 | 시간에 따른 시퀀스와 피로도 누적 학습 반영 |
| 이상 정의 | 사전에 라벨링된 고장 패턴과 일치하는가? | 학습된 정상 패턴과 비교하여 낯선 형태인가? |

- **오토인코더**: 입력 데이터를 그대로 출력으로 복원하도록 학습되는 비지도 학습 기반 신경망. 입력값과 출력값 사이의 재구성 오차(Reconstruction Error)를 최소화하는 방향으로 학습.
- **LSTM**: 순환 신경망의 한 종류로, 시계열 데이터 처리에 최적화.
- **LSTM-Autoencoder**: LSTM의 순차적 데이터 처리 능력 + 오토인코더의 데이터 압축 및 복원 능력 결합. 인코더와 디코더를 모두 LSTM 셀로 구성. 정상 데이터로 학습한 모델에 이상 데이터가 입력되면 정상 패턴으로 복원하지 못해 거대한 재구성 오차 발생 → 이상 탐지.

### II. AI4I 2020 Predictive Maintenance Dataset

**데이터셋 개요**:
- 총 10,000개 인스턴스, 14개 특성(Features), 결측치 0개
- 클래스 불균형: 정상 96% vs 고장 4% (Imbalance Ratio = 96:4)
- 실제 산업 장비의 마모, 온도, 회전 속도 등 물리적 특성을 수학적으로 모델링하여 생성한 인공 생성(Synthetic) 데이터

**주요 변수**:
- UDI: 고유 식별자 (1~10,000)
- Product ID: 제품 품질 등급 문자 + 일련번호
- Type: L(Low, 50%), M(Medium, 30%), H(High, 20%)
- Air temperature [K], Process temperature [K], Rotational speed [rpm], Torque [Nm], Tool wear [min]

**타깃 변수 (2단계 분류 구조)**:
- Level 1: Machine Failure — 이진 분류 (0: 정상 / 1: 고장). 하위 5가지 세부 고장 유형 중 하나라도 발생하면 '1' 반환.
- Level 2: 5대 세부 고장 유형 — 다중 레이블 분류 (각각 0 또는 1)
  - TWF (Tool Wear Failure): 공구 마모 고장
  - HDF (Heat Dissipation Failure): 방열 고장
  - PWF (Power Failure): 전력 고장
  - OSF (Overstrain Failure): 과부하 고장
  - RNF (Random Failures): 무작위 고장

**데이터 전처리 파이프라인 (4단계)**:
1. **[Step 1] 식별자 제거 및 누수 차단** (Noise & Leakage Removal): UDI, Product ID 같은 무의미한 고유 식별자 제거. 세부 고장 모드 변수(TWF, HDF, PWF, OSF, RNF)는 '고장 발생 이후'의 결과성 데이터이므로 입력 피처에서 철저히 배제 → Data Leakage 방지.
2. **[Step 2] 층화 추출 및 스케일링** (Stratification & Scaling): 층화 추출법(Stratified Sampling)으로 Train/Test 데이터셋 분리. 고장 비율을 Train과 Test 모두 동일하게 유지 (stratify=y 옵션). 범주형 Type 변수는 One-Hot Encoding 적용.
3. **[Step 3] Tensor 변환 및 차원 조정** (PyTorch Dataset): np.float64 → torch.float32 타입 변환. BCE Loss 계산을 위해 타깃 텐서에 .unsqueeze(1) 적용하여 [batch_size] → [batch_size, 1] 차원 조정.
4. **[Step 4] 오버샘플링 및 미니배치** (Weighted DataLoader): WeightedRandomSampler를 사용하여 희소한 고장 데이터가 더 높은 확률로 중복 추출되도록 설정 → 배치마다 고장 패턴 고르게 학습.

**모델 아키텍처 (MLP)**:
- Input Layer → Linear(32) + ReLU + BatchNorm1d + Dropout(0.3) → Linear(16) + ReLU + BatchNorm1d + Dropout(0.3) → Linear(1) [Logits Output]
- ReLU: 선형 연산의 한계를 극복, 비선형 패턴 학습 활성화 함수. R(z) = max(0, z)
- BatchNormalization: 각 배치별 데이터 분포 정규화, 학습 속도 가속화, internal covariate shift 방지
- Dropout(0.3): 매 훈련 스텝마다 30%의 노드 연결 차단 → 특정 뉴런 과의존 제거, 과적합 방지

**1 Epoch 학습 과정 (6단계)**:
1. DATA LOADING & BATCHING: 데이터 셔플 후 배치 크기로 나누어 GPU 전송
2. FORWARD PROPAGATION(순전파): 입력 데이터가 가중치(W)와 활성화 함수를 거쳐 예측값(ŷ) 계산
3. LOSS COMPUTATION(손실 계산): 예측값(ŷ)과 실제 정답(y) 비교, 오차(Loss, E) 스칼라 값으로 수치화
4. BACKWARD PROPAGATION(역전파): 연쇄 법칙(Chain Rule)으로 각 가중치의 기울기(Gradient, ∂E/∂W) 계산
5. WEIGHT UPDATE(가중치 업데이트): Optimizer가 가중치 수정 (W_new = W_old - η · ∂E/∂W)
6. EPOCH COMPLETE & EVALUATION: 검증 데이터로 일반화 성능 평가 및 기록

**Train/Validation/Test 데이터셋 역할**:
- Train: 모델의 가중치(Weight)를 직접 업데이트하며 학습 (교과서와 연습문제)
- Validation: 학습 도중 성능 평가, 하이퍼파라미터 튜닝 및 Early Stopping 결정 (모의고사)
- Test: 모든 학습이 끝난 후 단 한 번만 사용하는 성능 측정용 데이터 (수능 시험)

**평가 지표 (Confusion Matrix 기반)**:
- Accuracy(정확도): (TP + TN) / (TP + TN + FP + FN) — 전체 중 올바르게 예측한 비율
- Precision(정밀도): TP / (TP + FP) — 양성이라 예측한 것 중 실제 양성 비율
- Recall(재현율): TP / (TP + FN) — 실제 양성 중 모델이 양성으로 찾아낸 비율
- F1-Score: 2 × (Precision × Recall) / (Precision + Recall) — Precision과 Recall의 조화평균

**모델 배포 (R&BD Deployment Package)**:
- `.pth` 파일: PyTorch 모델 가중치(state_dict) 또는 모델 전체 구조 저장
- `.pkl` 파일: 전처리 스케일러(StandardScaler) 저장. 추론 시 학습 당시와 완전히 동일한 기준(평균, 분산)으로 스케일링해야 함 → 스케일러 없이 모델만 배포하면 정상적 추론 불가.

**실시간 추론 파이프라인 (4단계)**:
단계 1(객체 로드) → 단계 2(데이터 수집) → 단계 3(전처리) → 단계 4(AI 진단)

**모델 학습 vs 실시간 추론 비교**:
- 데이터 유입: 대규모 배치 CSV vs 초/분 단위 단일 건 스트림 데이터
- 연산 모드: Gradient 계산 및 가중치 업데이트 활성화 vs 가중치 고정, 역전파 비활성화
- 최우선 목표: Loss 최소화 및 정확도 최적화 vs 지연시간(Latency) 최소화 및 시스템 안정성 보장

## 🔑 키워드 정리

| # | 키워드 | 핵심 정의 |
|---|--------|-----------|
| 1 | 예지보전 (Predictive Maintenance) | 설비 고장이 발생하기 전, 데이터 분석으로 이상을 사전에 탐지하여 고장을 예방하는 유지보수 전략 |
| 2 | PLC (Programmable Logic Controller) | 제조 설비를 자동 제어하는 산업용 컴퓨터로, 센서 데이터를 실시간으로 수집·전송 |
| 3 | 오토인코더 (Autoencoder) | 입력을 잠재 벡터(Latent Vector)로 압축 후 재구성하도록 학습되는 비지도 학습 신경망 |
| 4 | LSTM (Long Short-Term Memory) | 시계열 데이터의 장기 의존성을 학습하는 순환 신경망(RNN)의 한 종류 |
| 5 | LSTM-Autoencoder | LSTM의 순차 처리 능력과 오토인코더의 압축·복원 능력을 결합한 시계열 이상 탐지 모델 |
| 6 | 재구성 오차 (Reconstruction Error) | 오토인코더가 입력을 복원했을 때 원본과 복원본 사이의 차이(오차). 이상 판단 기준 |
| 7 | 클래스 불균형 (Class Imbalance) | 데이터셋에서 특정 클래스(예: 고장)의 샘플 수가 다른 클래스(정상)에 비해 극단적으로 적은 상태 |
| 8 | 데이터 누수 (Data Leakage) | 모델 학습 시 미래 또는 결과 정보가 입력 피처에 포함되어 비현실적으로 높은 성능이 나오는 오류 |
| 9 | 층화 추출법 (Stratified Sampling) | 전체 데이터를 특정 기준(클래스 비율 등)에 따라 하위 그룹으로 나눈 뒤 각 그룹에서 무작위 표본을 추출하는 방식 |
| 10 | One-Hot Encoding | 범주형 데이터를 컴퓨터가 처리할 수 있도록 해당 범주만 1, 나머지는 0으로 표기하는 이진 벡터 변환 |
| 11 | WeightedRandomSampler | 클래스 불균형 상황에서 소수 클래스가 더 높은 확률로 샘플링되도록 가중치를 부여하는 PyTorch 기능 |
| 12 | Dropout | 훈련 시 무작위로 일정 비율의 뉴런을 비활성화하여 과적합을 방지하는 정규화 기법 |
| 13 | BatchNormalization | 미니배치 단위로 데이터 분포를 정규화하여 학습 안정화 및 internal covariate shift를 방지하는 기법 |
| 14 | ReLU (Rectified Linear Unit) | max(0, z)로 정의되는 비선형 활성화 함수. 선형 연산의 한계를 극복하고 비선형 패턴을 학습 |
| 15 | 역전파 (Backpropagation) | 연쇄 법칙(Chain Rule)을 이용하여 출력 오차에 대한 각 가중치의 기울기(Gradient)를 계산하는 알고리즘 |
| 16 | Epoch | 훈련 데이터셋에 포함된 모든 샘플이 네트워크를 통해 한 번씩 Forward + Backward 통과한 상태 |
| 17 | Early Stopping | 검증 손실(Validation Loss)이 증가하기 시작하는 교차점을 파악하여 과적합 이전에 학습을 조기 종료하는 기법 |
| 18 | Confusion Matrix | 모델이 어떤 종류의 오류를 내며 헷갈리고 있는지를 직관적으로 보여주는 N×N 형태의 교차표 |
| 19 | Recall (재현율) | 실제 양성(고장) 데이터 중 모델이 양성으로 올바르게 찾아낸 비율. TP / (TP + FN) |
| 20 | F1-Score | Precision과 Recall의 조화평균. 클래스 불균형 상황에서 Accuracy보다 신뢰할 수 있는 지표 |
| 21 | .pth 파일 | PyTorch 딥러닝 모델의 가중치(state_dict) 또는 전체 모델 구조를 저장하는 파일 형식 |
| 22 | .pkl 파일 | 파이썬 프로그램 간에 복잡한 데이터(예: 스케일러 객체)를 주고받거나 저장하는 파일 형식 |
| 23 | Synthetic Data | 실제 장비에서 직접 수집하지 않고, 물리적 특성과 고장 메커니즘을 수학적으로 모델링하여 인공적으로 생성한 데이터 |
| 24 | internal covariate shift | 학습 중 각 레이어의 입력 데이터 분포가 계속 변화하는 현상. BatchNormalization으로 방지 |
| 25 | TensorBoard | 머신러닝 실험을 시각화하고 모니터링하기 위한 웹 기반 도구 (SummaryWriter로 로그 기록) |

### 문제 2 `[심화]`

> 다음 문장이 옳으면 O, 틀리면 X를 고르시오.

**"LSTM-Autoencoder는 이상 데이터를 직접 학습하기 때문에 사전에 라벨링된 고장 패턴 없이는 이상 탐지가 불가능하다."** &nbsp;&nbsp; ( O / X )

> [!tip]- 💡 정답 및 해설 보기

>
> **해설**: 층화 추출법의 핵심은 전체를 층(Strata)으로 분리한 후 각 층에서 독립적으로 무작위 추출하는 것이다. 클래스 불균형 데이터에서 `stratify=y` 옵션을 사용하면 Train/Test 모두 원본 데이터의 클래스 비율을 정확히 유지할 수 있다.

### 문제 8 `[심화]`

**Train / Validation / Test 데이터셋의 역할이 잘못 설명된 것은?**

> ① Train 데이터: 모델의 가중치(Weight)를 직접 업데이트하며 학습하는 데이터
> ② Validation 데이터: 학습 도중 성능 평가를 통해 하이퍼파라미터 튜닝과 Early Stopping 결정에 활용되는 데이터
> ③ Test 데이터: 모든 학습이 끝난 후 단 한 번만 사용하여 최종 성능을 측정하는 데이터
> ④ Validation 데이터: 학습이 완료된 후 최종 모델을 선택하는 데 사용되며, 여러 번 반복 평가하여 최고 성능을 기록한다

> [!info]- 💡 정답 및 해설 보기
> **정답**: ④
>
> **해설**: Validation 데이터는 학습 **도중** 사용하는 것이 맞지만, ④의 설명 중 "여러 번 반복 평가하여 최고 성능을 기록한다"는 부분이 문제다. Validation 데이터를 기반으로 하이퍼파라미터를 반복 튜닝하면, 결국 Validation 데이터에도 간접적으로 과적합이 발생할 수 있다. 그래서 **Test 데이터는 최종 평가 시 단 한 번만** 사용해야 한다. Validation과 Test의 역할을 혼동하게 만드는 함정이다.

> **모범 답안**: Data Leakage는 모델 학습 시 실제 배포 환경에서는 알 수 없는 미래 또는 결과 정보가 입력 피처에 포함되는 오류다. 이는 훈련/검증 환경에서 비현실적으로 높은 성능을 만들어내지만, 실제 배포 시 모델이 전혀 작동하지 않는 치명적 결과를 초래한다. AI4I 2020에서 Data Leakage를 유발하는 변수는 5대 세부 고장 모드(TWF, HDF, PWF, OSF, RNF)다. 이 변수들은 '기계가 고장 난 이후'에 기록되는 사후 결과 데이터다. 시간 흐름으로 보면 Time T-1(센서 데이터) → Time T(고장 발생, Machine Failure=1) → Time T+1(세부 고장 모드 기록)이다. 따라서 이 변수들을 입력 피처로 사용하면, 모델이 센서 데이터의 물리적 패턴을 학습하는 것이 아니라 결과 변수의 존재 여부만 보고 '고장'임을 판단하게 된다.
>
> **채점 포인트**:
> - [ ] Data Leakage의 정의(미래/결과 정보가 입력 피처에 포함)를 정확히 서술하는가
> - [ ] 배포 시 모델이 작동하지 않는 치명적 결과를 언급하는가
> - [ ] TWF, HDF, PWF, OSF, RNF가 구체적으로 어떤 변수인지 설명하는가
> - [ ] 이 변수들이 '고장 발생 이후'의 사후 결과 데이터임을 시간 흐름으로 설명하는가

**용접 예지보전 프로젝트에서 전통적 지도학습과 LSTM-Autoencoder(비지도학습)를 요구 데이터, 시간적 흐름(Temporal Logic), 이상(Anomaly)의 정의 세 가지 측면에서 비교·분석하고, 실제 제조 현장의 데이터 특성을 고려했을 때 LSTM-Autoencoder가 더 적합한 이유를 종합적으로 논하시오.**

>
> 시간적 흐름 측면에서, 전통적 지도학습(일반적인 정형 데이터 분류 모델)은 각 시점의 데이터를 독립적으로 판단하여 용접 공정의 시간에 따른 피로도 누적이나 이상 징후의 점진적 발전을 포착하지 못한다. LSTM-Autoencoder는 인코더·디코더 모두 LSTM으로 구성하여 시간적 시퀀스와 동적 패턴 변화를 학습하고, 시계열 전체를 하나의 잠재 벡터로 압축 후 복원하여 단순 시점별 비교를 넘어서는 시계열 특징 추출이 가능하다.
>
> 이상 정의 측면에서, 지도학습은 사전에 라벨링된 고장 패턴과의 일치 여부로 이상을 정의하므로 학습하지 않은 새로운 유형의 고장을 탐지하지 못한다. LSTM-Autoencoder는 학습된 정상 패턴과 비교하여 '낯선 형태'를 이상으로 정의하므로, 미리 정의되지 않은 새로운 유형의 이상도 탐지 가능하다는 장점이 있다.
>
> **핵심 채점 포인트**:
> - [ ] 세 가지 비교 기준(요구 데이터, 시간적 흐름, 이상 정의)을 모두 논하는가
> - [ ] 실제 제조 현장의 고장 데이터 희귀성과 불균형 문제를 구체적으로 언급하는가

> - [ ] [가점] 미지의 새로운 고장 유형 탐지 가능성(Open-set Anomaly Detection) 측면을 언급하는가

### 문제 2 `[심화]`

**AI4I 2020 모델에서 적용된 세 가지 정규화/제어 기법(ReLU, BatchNormalization, Dropout)이 각각 어떤 문제를 해결하며, 심층 신경망 설계에서 이 세 가지를 함께 사용하는 이유를 각 기법의 역할과 상호 보완 관계를 중심으로 논하시오.**

> [!example]- 💡 정답 및 채점 포인트
> **모범 답안**:
>
> ReLU(Rectified Linear Unit, max(0,z))는 선형 연산만으로는 학습할 수 없는 복잡한 비선형 패턴을 신경망이 표현할 수 있게 하는 활성화 함수다. 또한 Sigmoid 등 기존 활성화 함수의 기울기 소실(Vanishing Gradient) 문제를 완화하여 깊은 신경망의 역전파 학습을 안정화한다.
>
> BatchNormalization은 학습이 진행되면서 각 레이어의 입력 분포가 지속적으로 변화하는 internal covariate shift 문제를 해결한다. 각 미니배치 단위로 데이터를 평균 0, 분산 1로 정규화하여 학습 속도를 가속화하고 높은 학습률(Learning Rate)을 사용할 수 있게 해준다.
>
> Dropout은 훈련 시 무작위로 일정 비율의 뉴런을 비활성화하여 특정 뉴런에 대한 과의존을 방지하고 과적합(Overfitting)을 직접적으로 억제한다.
>
> 세 기법은 상호 보완적이다. ReLU는 표현력을, BatchNormalization은 학습 안정성과 속도를, Dropout은 일반화 성능을 각각 담당한다. BatchNormalization이 각 배치의 분포를 안정화하면 ReLU의 기울기 소실 문제가 더욱 완화되고, Dropout이 과적합을 방지하면서 모델이 다양한 피처 조합을 학습하도록 강제하여 결과적으로 더 강건한(Robust) 모델이 만들어진다.
>
> **핵심 채점 포인트**:
> - [ ] ReLU가 해결하는 문제(비선형성, 기울기 소실)를 정확히 설명하는가
> - [ ] BatchNormalization이 해결하는 문제(internal covariate shift)를 정확히 설명하는가
> - [ ] Dropout이 해결하는 문제(과적합, 뉴런 과의존)를 정확히 설명하는가
> - [ ] 세 기법의 상호 보완 관계를 논리적으로 서술하는가
> - [ ] [가점] BatchNorm과 Dropout을 함께 쓸 때 발생할 수 있는 순서 문제(Dropout before/after BatchNorm)를 언급하는가

### 문제 4 `[심화]`

**제조 AI 개발 파이프라인에서 '모델 학습(Training) 환경'과 '실시간 추론(Real-Time Inference) 환경'은 데이터 유입 형태, 연산 모드, 최우선 목표가 근본적으로 다르다. 두 환경의 차이를 세 가지 관점에서 비교하고, 이러한 차이가 실제 시스템 설계에 어떤 시사점을 주는지 논하시오.**

> [!example]- 💡 모범 답안 및 채점 포인트
> **모범 답안**:
>
> 첫째, 데이터 유입 형태에서 학습 환경은 수집된 대규모 정적 CSV 데이터셋을 배치(Batch) 단위로 처리한다. 반면 추론 환경에서는 PLC/OPC-UA 서버로부터 초·분 단위로 들어오는 단일 건의 실시간 스트림 데이터를 즉시 처리해야 한다. 이는 데이터 로딩 방식, 전처리 파이프라인, 메모리 관리 전략이 완전히 달라야 함을 의미한다.
>
> 둘째, 연산 모드에서 학습 환경은 기울기(Gradient) 계산과 역전파를 통한 가중치 업데이트가 핵심이다. 추론 환경에서는 가중치가 고정(Frozen)되고 역전파가 비활성화(torch.no_grad())되어 순전파 연산만 수행한다. 이는 불필요한 기울기 계산을 제거하여 연산 속도를 크게 높이기 위함이다.
>
> 셋째, 최우선 목표에서 학습 환경은 Loss 최소화와 정확도 최적화가 목표다. 추론 환경에서는 지연시간(Latency) 최소화와 시스템 안정성 보장이 목표다. 공정 이상이 발생했을 때 수 초 내로 경보를 발령해야 하므로 추론 속도가 매우 중요하다.
>
> 이러한 차이는 시스템 설계에 중요한 시사점을 준다. 학습과 추론을 별도의 서버/환경으로 분리하고, 추론 서버는 torch.no_grad()와 model.eval()로 Dropout·BatchNorm 동작을 추론 모드로 전환해야 하며, 스케일러와 모델을 메모리에 미리 적재(Pre-loading)하여 실시간 요청마다 디스크에서 재로딩하는 지연을 없애야 한다.
>
> **핵심 채점 포인트**:
> - [ ] 데이터 유입 형태의 차이(대규모 배치 vs 단일 건 스트림)를 구체적으로 설명하는가
> - [ ] 연산 모드의 차이(역전파 활성/비활성)와 그 이유를 설명하는가
> - [ ] 최우선 목표의 차이(Loss 최소화 vs Latency 최소화)를 설명하는가
> - [ ] 실제 시스템 설계에 대한 시사점(torch.no_grad(), model.eval(), Pre-loading 등)을 언급하는가
> - [ ] [가점] Dropout과 BatchNorm이 추론 시 model.eval()에 의해 동작 방식이 달라짐(BatchNorm이 학습 통계치 사용)을 언급하는가
