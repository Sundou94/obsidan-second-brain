### I. 장비이상 조기탐지 AI — 열풍건조 설비

**열풍건조(Heat Drying) 공정의 배경**
- 도금 처리 이후 피막과 안정제가 금속 표면에 달라붙도록 하며, 탈수로 제거된 물기를 완전히 건조시키는 공정
- 설비 가동에 높은 전력이 소모되며, 내부 FAN 모터는 사전 설정된 적정값으로 회전하면서 고유 진동수 및 진동 패턴을 유지
- 코일 문제 등 이상 발생 시 고유 상태가 깨지면서 이상 현상 발생 → 소리 데이터로 이상 조기 탐지

**소리 데이터 수집 현황**
- 수집 기간: 2021.09.06 ~ 2021.10.27, 수집 주기: 5초
- FAN Sound 정상: 170개 / FAN Sound 불량: 13개 (극심한 데이터 불균형)

**Feature Extraction 필요성**
- 소리는 시간에 따른 파동(Wave) 형태 — 원본 음원을 모델에 직접 입력하면 연산량이 기하급수적으로 증가하고 핵심 정보 식별이 어려움
- 따라서 소리의 '핵심 정보'를 압축·추출하여 수치형 데이터로 변환하는 Feature Extraction 과정이 필수

**소리 데이터 처리 4단계 파이프라인 (librosa 활용)**
1. **Time Domain**: 원본 사운드 파형 — 시간에 따른 진폭 시각화. 정상/이상 파형은 육안으로 구분이 매우 어려움
2. **Frequency Domain (FFT)**: 신호를 주파수 성분으로 분해 → 각 주파수별 강도(Magnitude) 확인 가능. Half Spectrum을 사용(FFT 결과는 대칭 구조이므로 절반만 사용하여 중복 제거)
3. **Perceptual Domain (MFCC)**: 인간의 청각 특성을 반영하여 특징 추출. Mel-Scale Filter Bank를 통과시켜 저주파에 민감하고 고주파에 둔감한 비선형 특성을 모방
4. **Evaluation & Selection**: 상관분석 및 후진 소거법으로 변수 최적화

**FFT (Fast Fourier Transform)**
- 시간 축 신호를 주파수 축으로 변환하여 각 주파수 성분의 강도를 확인
- Half Spectrum: FFT 결과는 대칭 구조이므로 중복을 피하기 위해 주파수의 절반만 사용

**MFCC (Mel Frequency Cepstral Coefficient)**
- 인간의 달팽이관은 저주파(낮은 소리) 변화에 매우 민감하지만, 고주파(높은 소리) 변화는 잘 구분하지 못하는 비선형적 특성
- Mel-Scale Filter Bank를 Raw Frequency Spectrum에 적용하여 청각적 특징을 수치 벡터로 압축

**상관분석 및 후진 소거법**
- 상관분석: 두 변수 간 선형 또는 비선형 관계의 강도를 -1 ~ 1 사이 값으로 측정
- 후진 소거법(Backward Elimination): 상관관계가 낮은 변수를 제거하여 모델 성능을 높이는 기법
- 예: `mfcc_max` 변수는 다른 변수들과의 상관관계가 낮아(0.084, -0.24, -0.035) 후진 소거법으로 제거 대상이 됨

**Decision Tree**
- 데이터의 속성(Feature)을 기준으로 질문을 던져가며 분류(Classification) 또는 회귀(Regression)를 수행하는 기계학습 알고리즘
- 노드 구성: Root Node(전체 데이터 시작점) → Internal/Decision Node(분할 기준) → Leaf/Terminal Node(최종 분류 결과)
- **Entropy**: 데이터의 불순도(Impurity) 또는 불확실성(Uncertainty)을 측정. 동일 클래스만 존재 시 0, 50:50 혼합 시 1
- **Information Gain**: 분할 전 엔트로피 - 분할 후 자식 노드들의 엔트로피 가중평균. 모델은 Information Gain이 최대가 되는 방향으로 노드를 분할
- Decision Tree는 **화이트박스 모델** — 결과 해석이 용이하여 산업 현장에서 엔지니어가 납득할 수 있는 근거 제공. 적은 데이터에서도 유의미한 패턴 추출 가능

**Confidence 기반 시스템 관리**
- `model.predict_proba` 함수로 신뢰도(Confidence) 산출
- 95% 이상: 이상 탐지 확실 → Auto-alarm 트리거
- 70% ~ 95%: 불확실성 구간(Human-in-the-loop) → 현장 작업자에게 육안 및 청각 검사 요청
- 70% 미만: 정상 판단 → 시스템 정상 가동 유지

### II. MIMII 데이터셋 기반 AI 개발

**MIMII (Malfunctioning Industrial Machine Investigation and Inspection)**
- 실제 제조 현장의 다양한 산업용 기계(밸브, 펌프, 팬, 슬라이더 등)의 정상/비정상 소음 데이터를 수집한 오픈소스 데이터셋
- 데이터 불균형 상황에서 기계의 이상 징후를 사전 포착하는 **예지보전 AI 연구의 표준 벤치마크**
- 8채널 마이크로폰 어레이(직경 68mm, 45도 간격 배치)로 수집

**데이터 변환 파이프라인 (MIMII)**
1. 시간 영역 (1D Array): 원본 파형
2. 주파수 영역 (2D Matrix): STFT 적용 → Spectrogram
3. 텐서 영역 (2D Tensor): Mel Filterbank 적용 → Mel-Spectrogram

**STFT (Short Time Fourier Transform)**
- 시간에 따라 어떤 주파수 성분을 가지는지 추출
- 2차원 평면(시간 × 주파수)에 나타낸 스펙트로그램 도출
- 단순한 진폭 변화가 아닌, **특정 시간에 발생하는 이상 주파수 패턴**을 학습 가능

**Spectrogram vs Mel-Spectrogram**
- Spectrogram: STFT를 거쳐 나온 결과물 (선형 주파수 축)
- Mel-Spectrogram: STFT 수행 후 Mel Filterbank를 곱하여 저주파/고주파 특성을 비선형적으로 반영. 인간의 청각 인지를 모방하고 고주파 노이즈를 억제하는 장점

**CNN 기반 오토인코더 (Convolutional Autoencoder)**
- 2D 이미지(Mel-Spectrogram 128×128)를 입력으로 학습
- 구조: Encoder(Conv2d → ReLU → BatchNorm) → Latent Space(64-dim) → Decoder(ConvTranspose2d → ReLU → BatchNorm → Sigmoid)
- **핵심 아이디어**: 정상 데이터만으로 학습 → 정상 패턴을 복원하는 방법을 학습
- 정상 입력 시: 낮은 MSE (복원 성공) → 정상 판정
- 비정상 입력 시: 처음 보는 낯선 주파수 패턴을 '정상' 형태로 강제 변환 → 이상 발생 부위 유실 → 높은 MSE → 이상 판정

**복원 오차(MSE) 분포 분석**
- 두 분포(정상/비정상)가 멀리 떨어질수록(Separability가 높을수록) 모델의 특징 추출 능력이 우수
- 분포가 겹치는 구간(Overlap Area)은 오탐지 발생 구간 → Latent Space 최적화 또는 전처리 개선이 필요하다는 신호

**Confusion Matrix 해석 (예지보전 관점)**
- True Negative(정상→정상): 정상 가동 유지
- False Positive(정상→비정상): 불필요한 점검 (비용 낭비)
- **False Negative(비정상→정상): 치명적인 고장 누락 (가장 위험)**
- True Positive(비정상→비정상): 성공적인 예지보전

### 문제 2 `[기본]`

> 다음 문장이 옳으면 O, 틀리면 X를 고르시오.

**"FFT(Fast Fourier Transform) 결과는 대칭 구조로 나오기 때문에, 전체 주파수 스펙트럼을 모두 사용하는 것이 정보 손실을 최소화하는 올바른 방법이다."** &nbsp;&nbsp; ( O / X )

> [!tip]- 💡 정답 및 해설 보기
> **정답**: X
>

> **정답**: ③
>
> **해설**: ①번 오답 — 두 분포가 겹친다는 것은 오히려 구분이 잘 안 된다는 의미이다. ②번 오답 — 수집 문제가 아니라 모델링/전처리 개선의 문제일 수 있다. ③번 정답 — 강의에서 명시적으로 제시된 연구적 단서이다. ④번 오답 — 지도학습 교체가 확정적 해결책이 아니며, 먼저 Latent Space 최적화와 전처리를 개선하는 것이 올바른 접근이다.

**소리 데이터의 Feature Extraction 4단계 파이프라인(Time Domain → Frequency Domain → Perceptual Domain → Evaluation & Selection)의 각 단계를 설명하고, 각 단계에서 이전 표현 방식의 한계를 극복하는 방법을 논하시오.**

> **모범 답안**:
>
> 1단계 Time Domain은 librosa로 원본 사운드 파형을 시간에 따른 진폭으로 시각화한다. 그러나 시간 축 파형만으로는 정상과 이상 팬의 미세한 패턴 차이를 육안이나 모델로 구분하기 극히 어렵다는 한계가 있다.
>
> 2단계 Frequency Domain에서는 FFT를 적용하여 신호를 주파수 성분으로 분해한다. 정상과 이상의 주파수별 에너지 분포 차이가 드러나며, Half Spectrum 기법으로 중복 데이터를 제거한다. 그러나 FFT는 전체 신호의 평균적 주파수 분포만 보여주며, 어떤 시점에서 이상이 발생했는지 시간 정보가 손실된다.
>
> 3단계 Perceptual Domain에서는 MFCC를 적용하여 인간 청각의 비선형 특성을 반영한다. Mel-Scale Filter Bank가 저주파에 세밀하고 고주파에 거칠게 반응하도록 설계되어, 기계 이상 탐지에 중요한 저주파 변화를 더 민감하게 포착한다.
>
> 4단계 Evaluation & Selection에서는 상관분석으로 변수 간 관계를 파악하고, 후진 소거법으로 상관관계가 낮은 변수(예: mfcc_max)를 제거하여 모델 입력 차원을 최적화한다. 이를 통해 불필요한 변수로 인한 과적합과 연산 낭비를 방지한다.
>
> **핵심 채점 포인트**:
> - [ ] 각 4단계의 명칭과 핵심 기법(librosa, FFT/Half Spectrum, MFCC/Mel Filter Bank, 상관분석/후진소거법) 정확히 연결
> - [ ] Time Domain의 한계: 시각적 구분 불가 → FFT로 주파수 분해의 필요성 논리적 연결
> - [ ] FFT의 한계: 시간 정보 손실 및 비선형 청각 특성 미반영 → MFCC의 필요성 논리적 연결
> - [ ] Evaluation & Selection 단계에서 후진 소거법의 역할과 효과 설명
> - [ ] 가점: mfcc_max의 낮은 상관관계(-0.24, -0.035)를 구체적 수치로 언급하며 후진 소거 대상으로 설명

**CNN 기반 오토인코더를 활용한 MIMII 이상 탐지 시스템의 전체 파이프라인을 설명하고, MSE 분포의 Separability가 낮을 때(두 분포가 겹칠 때) 연구자가 취할 수 있는 개선 전략을 두 가지 이상 제시하고 각각의 근거를 논하시오.**

> **모범 답안**:
>
> **전체 파이프라인**: 원본 소리 파형(1D Array) → STFT 적용 → Spectrogram(2D Matrix) → Mel Filterbank 적용 → Mel-Spectrogram(128×128 2D Tensor) → CNN Encoder(Conv2d→ReLU→BatchNorm×3) → Latent Space(64-dim 압축) → CNN Decoder(ConvTranspose2d→ReLU→BatchNorm→Sigmoid) → 복원 이미지(128×128) → MSE 계산 → 임계값 비교 → 정상/이상 판정.

>

>
> 전략 2 — 전처리(소음 필터링) 강화: 산업 환경에는 배경 소음(환경 소음, 다른 설비 소음)이 섞여 정상/비정상 신호의 MSE 분포를 흐릴 수 있다. Band-pass Filter나 노이즈 게이팅 등의 전처리로 관심 주파수 대역을 강조하고 불필요한 노이즈를 제거하면 두 분포의 분리가 개선될 수 있다.
>
> 전략 3 (가점) — 입력 표현 변경: Spectrogram 대신 Mel-Spectrogram, 또는 다양한 window size와 hop length 실험을 통해 더 유의미한 주파수-시간 표현을 찾는다.
>
> **핵심 채점 포인트**:
> - [ ] 전체 파이프라인 단계(1D→STFT→Mel-Spectrogram→Encoder→Latent→Decoder→MSE)를 순서대로 정확히 서술
> - [ ] Separability의 의미(두 분포 간 거리)와 낮을 때의 문제(오탐지 발생)를 정확히 이해
> - [ ] Latent Space 최적화 전략과 근거를 논리적으로 서술
> - [ ] 전처리(소음 제거) 전략과 근거를 논리적으로 서술
> - [ ] 가점: 3번째 전략 또는 데이터 증강, 앙상블 등 추가 제안과 근거 제시

### 문제 3 `[심화]`

**제조 현장에서 소리 데이터 기반 AI 이상 탐지 시스템 도입 시, 데이터 불균형(정상 데이터 >> 이상 데이터)이 모델 개발과 평가에 미치는 영향을 분석하고, 이를 극복하기 위한 전략을 Confusion Matrix 관점에서 논하시오.**

> [!example]- 💡 모범 답안 및 채점 포인트
> **모범 답안**:
>
> 제조 현장에서는 이상 발생 자체가 드물기 때문에 데이터 불균형이 필연적으로 발생한다(이 강의의 사례: 정상 170개 vs 불량 13개, 약 13:1 비율). 이런 불균형 데이터로 지도학습 모델을 학습시키면, 모델이 모든 샘플을 정상으로 예측해도 높은 정확도(Accuracy)를 얻을 수 있어 평가 지표가 왜곡된다. 예를 들어, 모든 샘플을 정상으로 예측하면 정확도는 170/(170+13) ≈ 92.9%에 달하지만, 실제 불량을 하나도 잡지 못한다.
>
> Confusion Matrix 관점에서 가장 중요한 지표는 Recall(재현율, TP/(TP+FN))이다. 예지보전에서는 실제 비정상을 놓치는 False Negative가 치명적이므로, 정확도보다 Recall을 극대화하는 것이 목표가 되어야 한다. 그러나 Recall을 높이면 False Positive가 늘어나 불필요한 점검 비용이 증가하는 트레이드오프가 발생한다.
>
> 극복 전략: ① 오토인코더 활용 — 이 강의에서 채택한 방법으로, 정상 데이터만으로 학습하여 불균형 문제를 근본적으로 회피한다. ② 데이터 수준의 대응 — 불량 데이터 증강(SMOTE, 합성 데이터 생성) 또는 정상 데이터 다운샘플링으로 균형을 맞춘다. ③ Confidence 기반 운영 — 불확실 구간(70~95%)에 Human-in-the-loop을 도입하여 FN의 실제 발생을 현장에서 2차로 차단한다.
>
> **핵심 채점 포인트**:
> - [ ] 데이터 불균형의 구체적 비율(170:13)과 이것이 모델 학습에 미치는 영향(정확도 왜곡) 설명
> - [ ] Accuracy 지표의 한계와 Recall의 중요성을 Confusion Matrix로 연결
> - [ ] False Negative의 위험성을 예지보전 목표와 연결
> - [ ] 오토인코더 활용이 불균형 문제를 근본적으로 회피하는 방법임을 설명
> - [ ] 가점: Precision-Recall 트레이드오프 또는 F1-Score 언급, Human-in-the-loop의 역할 통합 논의
