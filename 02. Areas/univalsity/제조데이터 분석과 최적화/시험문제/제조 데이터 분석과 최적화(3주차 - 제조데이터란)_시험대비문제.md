### I. 제조데이터란?

제조데이터는 제품의 기획, 설계, 생산, 유통, 폐기 및 재활용에 이르는 **제품 수명 주기(Product Lifecycle) 전반**에서 생성되는 모든 형태의 디지털 정보를 총칭한다. 제조데이터는 제조공정 상 효율적인 제품 개발·생산을 가능하게 하는 **원동력이자 핵심 자원**으로 정의된다.

산업 현장에 AI를 도입하는 일반적인 프로세스는 다음 7단계로 구성된다:
**문제정의 → 데이터 수집 → 전처리 → 모델링 → 검증 → 분석 → 적용**

이 중 **데이터 수집과 전처리** 단계가 핵심으로 강조되며, 제조 AI 프로젝트의 **87%가 PoC(Proof of Concept) 단계에서 실패**하는 주요 원인은 테스트 환경과 실제 제조 환경의 **데이터 불일치(Data Mismatch)** 및 생산 환경의 복잡성을 충분히 고려하지 못한 데 있다.

### II. 제조 데이터의 원천: 4M1E

제조 데이터를 수집하기 전에 **전체 생산 프로세스를 이해**하는 것이 필수다. 제조 현장을 구성하는 **5대 핵심 요소(4M1E)**는 다음과 같다:

- **Man(사람)**: 작업자, 관리자, 기술자 등 — 숙련도, 피로도, 작업 실수 기록
- **Machine(기계/설비)**: 모든 장비 및 설비 — 가동률, 고장 빈도, 센서 실시간 데이터(온도·압력·진동)
- **Material(재료)**: 원자재, 부품, 소모품 — LOT 추적 정보, 재고 수준, 품질 검사 결과
- **Method(방법)**: 표준 작업 절차(SOP), 공정 흐름 — 공정 매개변수 설정값, 최적화 알고리즘
- **Environment(환경)**: 작업장의 물리적 조건 — 온도, 습도, 조명, 청정도, 소음 수준

### III. 제조 현장의 암묵지(Tacit Knowledge)

**암묵지(Tacit Knowledge)**란 학습과 경험으로 개인에게 체화됐지만 언어나 문자로 표현될 수 없어 겉으로 드러나지 않는 지식이다. 이는 **형식지(Explicit Knowledge)**(문서, 매뉴얼 등 공유 가능한 지식)와 대비된다.

제조 현장 암묵지의 특징: **비정형**(매뉴얼·DB로 기록 불가), **내재화**(특정 작업자에 의존), **전수 어려움**(은퇴 시 소멸 위험).

암묵지를 형식지로 변환하는 **SECI 모델** 4단계:
1. **사회화(Socialization)**: 암묵지 → 암묵지 (숙련공과 함께 작업하며 경험 전수)
2. **외재화(Externalization)**: 암묵지 → 형식지 (영상, 소리, 동작 데이터로 변환)
3. **결합(Combination)**: 형식지 + 형식지 (디지털화된 노하우를 MES 데이터와 결합)
4. **내재화(Internalization)**: 형식지 → 암묵지 (작업자가 디지털 지식을 다시 체득)

제조업의 AI 융합 혁신 핵심: **현장 암묵지를 AI가 학습할 수 있는 형식지(Explicit Knowledge)로 변환**하는 것.

### IV. 제조데이터의 특성: 데이터 유형

| 유형 | 특징 | 예시 |
|------|------|------|
| **정형 데이터(Structured)** | 고정된 스키마, 행·열 테이블 구조 | RDBMS, 센서 로그 CSV |
| **반정형 데이터(Semi-structured)** | 고정 스키마 없음, 메타데이터/태그로 구조 내포 | JSON, XML, HTML |
| **비정형 데이터(Unstructured)** | 미리 정의된 모델·스키마 전혀 없음, Raw 그대로 | 이미지, 영상, 음성, 텍스트 |

- **정형 데이터**를 다루는 표준 언어: **SQL(Structured Query Language)**. DBMS(Database Management System)를 통해 관리.
- **반정형 데이터**: 2000년대 후반 인터넷 서비스 폭발로 중요성 급부상. JSON, XML, HTML이 주류. **Parsing**(문법 규칙에 따른 구조적 분석)을 통해 기계적 해석 가능.
- **비정형 데이터**: 전체 데이터의 약 **80%** 차지(비정형 중 전자문서 32%, SNS 25%, IoT 23%). **Data Lake** 또는 **Object Storage**에 원본 그대로 저장.

### V. 제조 데이터셋과 형태

제조 데이터 수집 및 활용 목적: **생산성 향상(공정 최적화), 품질 향상(불량 검출), 원가 절감/비용 최적화(설비 예지보전)**, 그리고 **디지털 전환(DX)·AI 전환(AX)**을 통한 스마트공장 구현.

**주요 제조 데이터셋 유형:**

- **Audio 데이터**: 기계 이상음 진단에 활용. 오디오는 1차원 파형(Waveform). 기계 고장 정보는 주파수 영역(Frequency Domain)에 숨어 있으므로 **STFT(Short-Time Fourier Transform)**를 통해 **스펙트로그램(Spectrogram)**으로 변환하여 학습. 대표 데이터셋: **MIMII(Malfunctioning Industrial Machine Investigation and Inspection)** — 펌프, 밸브, 팬, 슬라이드 레일 4종.

- **이미지 데이터**: 컴퓨터 비전(Computer Vision) 기반 표면 결함 검사. 4가지 과제: 분류(Classification), 객체 탐지(Object Detection), 세그멘테이션(Segmentation), **이상 탐지(Anomaly Detection)**. 제조에서는 불량 데이터 희소성으로 정상 이미지만 학습하는 **비지도 학습(Unsupervised Learning)** 방식 선호. 대표 데이터셋: **MVTec AD** — 객체·텍스처 15개 카테고리.

- **시계열 데이터**: 센서들이 일정 간격으로 기록하는 **연속적인 수치(Sequence) 흐름**. 핵심은 단편적 수치가 아닌 시간 축의 **추세(Trend), 주기(Cyclical), 계절성(Seasonal), 이상치(Anomaly)** 패턴 파악. 대표 데이터셋: **CWRU(Case Western Reserve University) Bearing Dataset** — 내륜/외륜/전동체 결함 분류.

### VI. 데이터셋: 오픈소스 vs 비공개

| 구분 | 오픈소스 데이터셋 | 비공개(Closed) 데이터셋 |
|------|------------------|------------------------|
| 주요 목적 | 알고리즘 연구, 벤치마킹, 교육용 | 공정 최적화, 예지보전 등 실무 적용 |
| 데이터 상태 | 비교적 깔끔, 변수 비식별화 | 노이즈 많음, 불균형 상태 |
| 보안/민감도 | 누구나 접근 가능 | 기업 핵심 영업비밀 |
| 도메인 결합도 | 낮음 (일반화 패턴) | 매우 높음 (특정 공장/설비 최적화) |
| 구축 비용 | 무료 또는 저렴 | 데이터 정제·레이블링에 막대한 비용 |

**주요 국내 오픈 플랫폼:**
- **AI Hub**: 과기부 주관, NIA 운영. 전 산업 범용 비정형 데이터 위주. 고비용/고품질 라벨링 데이터.
- **KAMP**: 중기부 주관, KAIST 협력. 중소 제조기업 특화. 설비 센서 시계열 수치 데이터(정형) 위주. 클라우드 기반 AI 분석 환경 및 AI 표준 모델 제공.

### VII. 개발환경 구축

AI 개발 환경 스택(하위 → 상위):
**GPU Card → Driver → OS → CUDA Toolkit → cuDNN → Tensorflow / PyTorch / ONNX Runtime**

- **CUDA(Compute Unified Device Architecture)**: Nvidia GPU 기반 병렬 컴퓨팅 아키텍처이자 프로그래밍 모델
- **cuDNN(CUDA Deep Neural Network library)**: 딥러닝 모델(Convolution, Activation, Pooling, Back Propagation 등) 연산 GPU 가속 라이브러리
- **ONNX Runtime**: 다양한 딥러닝 프레임워크에서 개발된 모델을 실행하는 실행엔진
- **Conda 가상환경**: 라이브러리 간 충돌·버전 문제를 방지하기 위해 프로젝트별 독립된 Python 환경 구축
- **Jupyter Notebook**: 셀 유형 — Code(Python 실행), Markdown(문서 작성), Raw(순수 텍스트). 셀 모드 — 에디터 모드([Enter]) vs 커맨드 모드([Esc]).

### VIII. 데이터셋 품질 확보

데이터셋 품질의 **5대 기준(MDQI)**:

| 기준 | 의미 |
|------|------|
| **유일성(Uniqueness)** | 완전히 동일한 중복 레코드 존재하지 않아야 함 |
| **완전성(Completeness)** | 핵심 속성에 누락된 값(결측치)이 없어야 함 |
| **유효성(Validity)** | 사전 정의된 도메인(규칙, 범위, 형식) 준수 |
| **일관성(Consistency)** | 데이터 간 논리적 관계가 모순 없이 앞뒤가 맞아야 함 |
| **정확성(Accuracy)** | 수집 데이터가 현실 세계의 실제 물리적 상태 반영 |

**MDQI(Manufacturing Data Quality Index)** 공식:
$$Q_i = \left(\frac{N - E}{N}\right) \times 100 \qquad MDQI = \frac{Q_{uni} + Q_{com} + Q_{val} + Q_{con} + Q_{acc}}{5}$$

| 등급 | MDQI 스코어 | 품질 상태 | AI 적용성 |
|------|------------|-----------|-----------|
| S(최우수) | 99점 이상 | Golden Data | 즉시 AI 투입 가능 |
| A(우수) | 95~99점 미만 | Good Data | 경미한 전처리 후 활용 |
| B(보통) | 80~95점 미만 | Fair Data | 집중 정제 필요 |
| C(미흡) | 80점 미만 | Poor Data | AI 학습 불가, 인프라 점검 요망 |

### IX. 독립변수와 종속변수

- **독립변수(Independent Variable, Feature, Input, X)**: 다른 변수에 영향을 받지 않고 독립적으로 변할 수 있는 값. 원인(Cause) 역할.
- **종속변수(Dependent Variable, Label, Target, Output, Y)**: 독립변수의 변화에 따라 결정되는 값. 결과(Effect) 역할. 최종 예측하거나 맞춰야 하는 정답.

### X. Pandas 핵심

- **Series**: 1차원 레이블이 붙은 배열 (열 하나)
- **DataFrame**: 여러 Series를 같은 인덱스로 모아 둔 2차원 표
- 데이터 살펴보기: `df.head()`, `df.tail()`, `df.shape`, `df.columns`, `df.info()`, `df.describe()`
- 슬라이싱: `df['컬럼명']`, `df[시작:끝]`, `.loc`, `.iloc`, `.at`

> - [ ] 비정형: 스키마 없음, Raw 데이터, Data Lake 저장 + 예시(이미지/영상/음성)

### 문제 4 `[심화]`

**MDQI(Manufacturing Data Quality Index)의 5대 기준을 나열하고, 각 기준이 제조 AI 관점에서 왜 중요한지 구체적인 사례를 들어 설명하시오.**

> [!example]- 💡 모범 답안 및 채점 포인트
> **모범 답안**: MDQI의 5대 기준은 유일성(Uniqueness), 완전성(Completeness), 유효성(Validity), 일관성(Consistency), 정확성(Accuracy)이다. ①**유일성** — 중복 레코드가 존재하면 모델이 특정 패턴을 과학습(Overfitting)하므로 반드시 제거해야 한다. ②**완전성** — 센서 결측값이 많으면 모델이 잘못된 패턴을 학습하여 예지보전 정확도가 떨어진다. ③**유효성** — 온도 센서가 물리적으로 불가능한 -999℃를 기록하면 유효성 위반이며 제거해야 한다. ④**일관성** — 동일 설비에 대해 시스템마다 서로 다른 단위(℃ vs K)를 사용하면 학습 불가. ⑤**정확성** — 센서가 실제 온도와 다른 값을 기록하면 아무리 많은 데이터라도 쓸모없다. 따라서 MDQI는 AI 모델 성능의 상한선을 결정한다.
>
> **채점 포인트**:
> - [ ] 5대 기준(유일성·완전성·유효성·일관성·정확성) 모두 정확히 나열
> - [ ] 각 기준에 대해 제조 현장 관점의 구체적 사례 제시
> - [ ] MDQI 등급(S/A/B/C)과 AI 적용성 연결 언급

### 문제 1 `[심화]`

**제조 현장의 암묵지(Tacit Knowledge)를 AI가 학습 가능한 형식지(Explicit Knowledge)로 전환하는 과정을 SECI 모델을 중심으로 논하고, 이 과정에서 발생할 수 있는 도전 과제와 해결 방향을 제시하시오.**

> [!example]- 💡 모범 답안 및 채점 포인트
> **모범 답안**:
>
> 제조업 AI 혁신의 핵심은 숙련 작업자의 암묵지를 AI가 학습할 수 있는 형식지로 변환하는 것이다. SECI 모델에 따르면 이 과정은 4단계로 이루어진다. **사회화(Socialization)** 단계에서 신입 작업자가 숙련공과 함께 작업하며 암묵지를 체득하고, **외재화(Externalization)** 단계에서 그 암묵지를 영상·센서 데이터·동작 캡처 등을 통해 형식지로 변환한다. **결합(Combination)** 단계에서는 디지털화된 노하우를 기존 MES 데이터·매뉴얼과 결합하여 더 완성된 형식지를 구성한다. 마지막 **내재화(Internalization)** 단계에서 작업자들이 이 디지털 지식을 다시 자신의 암묵지로 체득한다.
>
> 그러나 이 전환 과정에는 여러 도전 과제가 있다. 첫째, 암묵지는 정의상 언어화하기 어려우므로 어떤 행동·감각이 중요한 노하우인지 식별하는 것 자체가 어렵다. 둘째, 숙련공의 은퇴 등으로 외재화 기회가 사라질 수 있다. 셋째, 수집된 데이터가 암묵지를 충분히 반영하는지 검증하기 어렵다. 해결 방향으로는 멀티모달 센서(영상+음향+진동)를 통한 다각적 데이터 수집, 도메인 전문가와 데이터 과학자의 긴밀한 협업, 그리고 KAMP 같은 플랫폼을 통한 체계적 데이터 레이블링 파이프라인 구축을 제시할 수 있다.
>
> **핵심 채점 포인트**:
> - [ ] SECI 모델의 4단계(사회화-외재화-결합-내재화)를 정확히 설명하고 제조 현장 사례 연결
> - [ ] 외재화(Externalization)가 AI 도입의 핵심 단계임을 강조
> - [ ] 암묵지 전환의 도전 과제(식별 어려움, 은퇴로 인한 손실, 검증 어려움)를 구체적으로 기술
> - [ ] 해결 방향을 기술적(멀티모달 센서)·조직적(협업)·제도적(플랫폼) 측면에서 다각도 제시

### 문제 2 `[심화]`

**제조 AI 프로젝트에서 오픈소스 데이터셋(MIMII, CWRU, MVTec AD 등)과 기업 내 비공개 데이터셋의 특성 차이를 비교하고, 실제 제조 현장에 AI를 배포할 때 오픈소스 데이터셋으로 개발된 모델이 가질 수 있는 근본적인 한계와 이를 극복하기 위한 전략을 논하시오.**

> [!example]- 💡 모범 답안 및 채점 포인트
> **모범 답안**:
>
> 오픈소스 데이터셋(MIMII, CWRU, MVTec AD 등)은 알고리즘 연구와 성능 벤치마킹을 목적으로 구축되어 비교적 깔끔하게 정제되고 변수가 비식별화되어 있으며, 누구나 무료로 접근 가능하다. 반면 기업 내 비공개 데이터셋은 특정 공장·설비에 최적화된 고도메인 결합 데이터로, 노이즈가 많고 정상 데이터 위주의 불균형 상태이나 실무 적용성은 훨씬 높다.
>
> 오픈소스 기반 모델의 근본적 한계는 두 가지다. 첫째, **도메인 불일치(Domain Shift)**다. CWRU 데이터셋은 특정 사양의 2HP 모터 베어링에서 수집된 것이므로, 다른 제조사·사양·노후화 수준의 실제 설비에 바로 적용하면 성능이 급격히 저하된다. 둘째, 오픈소스 데이터는 **실제 생산 환경의 복잡성을 재현하지 못한다**. 실제 현장에는 다중 기계 간섭 소음, 환경 변수(온도·습도 변화), 인적 요인 등이 복합적으로 작용하지만, 오픈소스 데이터는 통제된 실험 환경에서 수집된 것이다. 이것이 제조 AI 프로젝트의 87%가 PoC 단계를 넘지 못하는 원인이다.
>
> 극복 전략으로는 ①**Transfer Learning**: 오픈소스로 사전 학습(Pre-training)한 후 기업 데이터로 Fine-tuning, ②**도메인 적응(Domain Adaptation)**: 소스 도메인(오픈소스)과 타깃 도메인(실제 현장) 간의 분포 차이를 줄이는 기법 적용, ③**데이터 수집 단계에서의 전체 생산 프로세스 이해**: 4M1E 프레임워크를 활용하여 실제 현장 복잡성을 반영한 데이터 설계, ④**KAMP·AI Hub 활용**: 검증된 오픈 제조 데이터와 사내 데이터를 결합하는 전략을 제시할 수 있다.
>
> **핵심 채점 포인트**:
> - [ ] 오픈소스 vs 비공개 데이터셋의 특성 차이를 주요 목적·데이터 상태·도메인 결합도·비용 측면에서 비교
> - [ ] 도메인 불일치(Domain Shift)를 핵심 한계로 정확히 제시
> - [ ] PoC 실패 87% 통계와 연결하여 실제 배포 시 한계 설명
> - [ ] 극복 전략을 기술적(Transfer Learning, Domain Adaptation) 및 데이터 수집 설계 측면에서 구체적으로 제시
> - [ ] (가점) KAMP·AI Hub 등 실제 플랫폼을 활용한 전략 제안

### 문제 3 `[심화]`

**정형 데이터, 반정형 데이터, 비정형 데이터가 제조 AI에서 각각 어떻게 활용되는지 구체적인 데이터셋 사례(MIMII, CWRU, MVTec AD 등)와 연결하여 설명하고, 전체 데이터의 80%를 차지하는 비정형 데이터에서 가치를 끌어내는 것이 왜 핵심 경쟁력이 되는지 논하시오.**

> [!example]- 💡 모범 답안 및 채점 포인트
> **모범 답안**:
>
> 제조 AI에서 세 가지 데이터 유형은 각각 다른 목적과 기술 스택으로 활용된다. **정형 데이터**는 센서에서 수집되는 시계열 수치 데이터로, CWRU Bearing Dataset이 대표적이다. 베어링의 진동 신호를 시계열로 기록하여 내륜/외륜/전동체 결함을 분류하는 예지보전(Predictive Maintenance) AI에 활용된다. SQL 기반 DBMS나 Pandas DataFrame으로 처리하며, KAMP에서 주로 제공하는 데이터 유형이다. **반정형 데이터**는 MES·ERP 시스템의 설비 운전 로그나 품질 검사 기록이 JSON/XML 형태로 기록되어, Parsing을 통해 공정 최적화 분석에 활용된다. **비정형 데이터**는 이미지(MVTec AD를 활용한 표면 결함 검사), 오디오(MIMII를 활용한 기계 이상음 진단)로, 컴퓨터 비전과 딥러닝으로 처리된다.
>
> 비정형 데이터 활용이 핵심 경쟁력인 이유는, 전체 엔터프라이즈 데이터의 약 80%가 비정형이기 때문이다. 정형 데이터만 분석하는 기업은 가용 데이터의 20%만 활용하는 셈이다. 특히 제조 현장의 암묵지—베테랑 작업자의 청각적 직관(기계음으로 고장 판단), 시각적 직관(표면 미세 결함 식별)—는 비정형 오디오·이미지 데이터로만 포착할 수 있다. STFT를 통한 스펙트로그램 변환, 비지도 학습 기반 이상 탐지 등 비정형 데이터 처리 기술을 보유한 기업만이 이 80%의 가치를 끌어낼 수 있으며, 이것이 제조 AI 혁신에서의 진정한 경쟁 우위다.
>
> **핵심 채점 포인트**:
> - [ ] 정형(CWRU, 시계열, 예지보전), 반정형(MES 로그, JSON/XML), 비정형(MIMII/오디오, MVTec AD/이미지)을 구체적 데이터셋과 연결하여 설명
> - [ ] 각 데이터 유형에 적합한 처리 기술(SQL/Pandas, Parsing, STFT/Computer Vision) 언급
> - [ ] 비정형 데이터가 전체의 80%임을 정량적으로 제시
> - [ ] 제조 현장 암묵지의 비정형적 특성과 연결
> - [ ] (가점) 비정형 데이터 처리 기술 보유 여부가 기업 AI 경쟁력을 결정한다는 논리 전개
