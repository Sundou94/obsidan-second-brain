# 필기 자료 만들기 (Note Taker)

학습용 옵시디언 마크다운 필기 자료를 생성하는 스킬이다.
업로드된 파일(PDF, PPTX 등)의 내용을 깊이 분석하여 체계적이고 실용적인 학습 노트를 만든다.

## 핵심 원칙

이 스킬의 목표는 단순한 내용 요약이 아니라, 학생이 **실제로 공부하고 복습하는 데 도움이 되는** 자료를 만드는 것이다.
좋은 필기 자료는 원본 슬라이드만 봐서는 놓칠 수 있는 맥락과 설명을 보완하며, 어려운 개념을 친절하게 풀어준다.

## 출력 형식: 옵시디언 마크다운 (.md)

반드시 아래 구조를 따라 마크다운 파일을 생성하고, `/sessions/.../mnt/outputs/` 폴더에 저장한다.
파일명은 한국어로 직관적으로 작성한다 (예: `4주차_제조데이터_수집및처리방법_필기.md`).

## 필기 자료 구조 (반드시 이 순서로 작성)

### 1. YAML 프론트매터 (메타데이터)

파일 최상단에 반드시 포함한다. 강의 자료에서 확인할 수 있는 모든 정보를 채운다.
**원본 페이지 정보도 반드시 포함한다.**

```yaml
---
title: "[강의명]"
subtitle: "[부제 또는 주제]"
subject: "[과목명]"
week: [주차 번호]
professor: "[교수명 (소속)]"
date_created: [오늘 날짜, YYYY-MM-DD 형식]
source_file: "[원본 파일명.pdf]"      # ← 원본 파일명 필수 기재
total_pages: [전체 페이지 수]         # ← 전체 페이지 수 필수 기재
tags:
  - [핵심 키워드1]
  - [핵심 키워드2]
  - [핵심 키워드3]
status: 학습중
---
```

### 2. 제목 및 목차

- H1 제목으로 강의 주제 명시
- 제목 바로 아래에 출처 인용 블록을 추가한다:

```markdown
> **출처**: [파일명].pdf | [교수명], [소속]
> **총 XX페이지** — 각 섹션 제목 옆에 `[p.XX]` 형식으로 원본 슬라이드 번호를 표기함
```

- 옵시디언 내부 링크 형식(`[[#섹션명]]`)으로 목차 구성
- 주요 섹션과 하위 섹션을 계층적으로 나열

### 3. 본문 내용

각 섹션을 다음 요소들을 활용하여 풍부하게 작성한다:

**원본 페이지 정보 (필수 포함)**:
- 모든 주요 섹션의 제목 바로 아래에 `> 📄 **원본 슬라이드**: p.XX` 형식으로 원본 페이지 번호를 반드시 표기한다.
- 페이지 범위가 여러 장인 경우 `p.XX–YY` 형식으로 표기한다.
- 특정 개념, 수식, 코드, 예제가 정확히 어느 페이지에서 나왔는지 `[p.XX]` 형식으로 인라인 표기도 적극 활용한다.
- 독자가 원본 슬라이드를 쉽게 찾아볼 수 있도록 페이지 추적을 철저히 한다.

예시:
```markdown
## 3. Iterative Bellman Equation

> 📄 **원본 슬라이드**: p.6–7

기본 Bellman Equation의 문제점 [p.6]:
...

### Deterministic State-Transition Case [p.7]
...
```

**표**: 비교, 분류, 구조화된 정보에 적극 활용

**코드 블록**: 흐름도, 계층 구조, 코드 예시 등을 표현할 때 사용

**옵시디언 Callout 블록** (중요하게 활용):
- `> [!tip] 용어 설명` — 어려운 전문 용어의 친절한 풀이 (★ 반드시 포함)
- `> [!info] 보충 설명` — 강의에서 다루지 않은 배경 지식이나 맥락 추가
- `> [!warning] 주의` — 자주 헷갈리는 개념이나 실수하기 쉬운 부분
- `> [!question] 의문점` — 추가로 알아봐야 할 내용이나 강의 중 생긴 의문

**용어 설명 원칙**:
- 강의 자료에 처음 등장하는 모든 전문 용어, 약어, 기술 용어는 `[!tip] 용어 설명` callout에서 쉬운 말로 설명한다
- "~란?" 형식으로 정의하고, 가능하면 일상적인 비유나 예시를 추가한다

**부가 설명 원칙**:
- 슬라이드에 짧게 적혀 있어 의미가 불명확한 내용은 `[!info]` callout으로 맥락을 보완한다
- 개념들 간의 연관 관계, "왜 이것이 중요한가"에 대한 설명을 추가한다

### 4. 내 생각 섹션 (반드시 포함)

학생이 자신의 생각을 기록할 수 있는 공간을 제공한다.

```markdown
## 🧠 내 생각 / 의문점

> *이 공간에 수업을 듣고 떠오르는 생각이나 의문점을 자유롭게 작성해 보세요*

### 💡 인사이트

-

### ❓ 의문점

- [강의 내용에서 자연스럽게 떠오를 수 있는 의문 2~3개를 예시로 적어준다]

### 🔗 연관 개념

- 이전 주차 내용과 연결되는 부분:
- 실제 적용 가능한 프로젝트 아이디어:
```

### 5. 시험 대비 핵심 정리 (반드시 포함)

시험에 나올 가능성이 높은 핵심 내용만 간결하게 정리한다.
강의에서 강조된 내용, 표로 정리된 비교 항목, 절차/순서, 정의 등을 우선적으로 선별한다.

```markdown
## 📝 시험 대비 핵심 정리

> **★★★ 반드시 암기할 내용**

### 1. [첫 번째 핵심 주제]
- 요점 1
- 요점 2

### 2. [두 번째 핵심 주제]
...
```

### 6. 퀴즈 (반드시 포함, 3~5개)

학습한 내용을 테스트할 수 있는 퀴즈를 작성한다.
- 시험에 실제로 나올 법한 형식으로 (객관식, 단답형, 서술형 혼합)
- 중요한 개념, 용어, 절차에 초점을 맞춘다
- 옵시디언의 접이식 정답 callout을 사용한다

```markdown
## ❓ 퀴즈

**Q1.** [문제]

① 선지1
② 선지2
③ 선지3
④ 선지4

> [!done]- 정답
> **[정답]** [간단한 해설]

---

**Q2.** [단답형 문제]

> [!done]- 정답
> [정답]
```

## 작업 절차

1. **파일 전체 읽기**: PDF는 페이지 범위를 나눠서 모두 읽는다 (최대 20페이지씩).
2. **페이지 매핑**: 각 섹션/개념이 몇 페이지에 있는지 메모해 두며 읽는다.
3. **내용 분석**: 강의의 핵심 주제, 목차 구조, 중요 개념을 파악한다.
4. **용어 목록 작성**: 강의 자료에서 추출한 전문 용어·약어 목록을 정리한다.
5. **마크다운 파일 작성**: 위의 구조를 따라 충실하게 작성한다. **모든 섹션에 원본 페이지 번호 표기 필수.**
6. **저장 및 전달**: `/sessions/.../mnt/outputs/` 에 저장하고 `present_files` 도구로 전달한다.

## 품질 기준

- **분량**: 짧으면 안 된다. 56페이지 분량의 강의라면 노트도 충분히 길어야 한다.
- **원본 페이지 추적**: 모든 주요 섹션에 `> 📄 **원본 슬라이드**: p.XX` 표기가 있어야 한다. **누락 시 불합격.**
- **용어 설명**: 처음 등장하는 기술 용어는 100% 설명해야 한다.
- **구조**: 목차, 계층 구조, 표가 있어야 한다. 단순 텍스트 나열은 불합격.
- **학습 활용도**: 원본 자료 없이 이 노트만으로도 개념을 이해할 수 있어야 한다.
- **퀴즈**: 단순 암기 문제가 아닌, 개념 이해를 묻는 문제를 만들어야 한다.

## 파일명 규칙

`[주차]_[주제]_필기.md` 형식으로 작성한다.
예: `4주차_제조데이터_수집및처리방법_필기.md`
