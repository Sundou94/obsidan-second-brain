## 밴딧 문제란

> [!note] 핵심 개념
> **밴딧 = 슬롯머신**. 여러 대의 슬롯머신 중 어느 것을 선택해야 최대한 많은 코인을 얻을 수 있는가?

### 문제 설정
- $n$개의 슬롯머신이 있으며, 각각 **승률(확률분포)이 다름**
- 플레이어는 정해진 횟수(예: 1,000회) 내에 플레이
- 처음에는 슬롯머신에 대한 정보가 없음 → **플레이하면서 좋은 것을 찾아야 함**
- **목표: 정해진 횟수 안에 최대의 코인 획득**

```
       Action (A)
Agent ────────────► Environment
(Player)           (Slot machine)
      ◄────────────
       Reward (R)
```

| 요소 | 내용 |
|------|------|
| **행동(Action)** | 하나의 슬롯머신을 선택하여 플레이 |
| **보상(Reward)** | 획득한 코인 수 |
| **상태(State)** | **정의되지 않음** ← 밴딧 문제의 특징! |

> [!tip] 왜 밴딧 문제를 먼저 배우는가?
> 상태(State)가 없는 **단순화된 강화학습 문제**이기 때문이다. [[03. 마르코프 결정 과정|MDP]]의 준비 단계로 이해하면 좋다.

## 행동 가치 (Action Value)

### 정의
행동 가치란 **어떤 행동을 취했을 때 얻을 수 있는 보상의 기댓값**이다.

$$Q(a) = \mathbb{E}[R \mid A = a]$$

- **확률분포**: 슬롯머신 $a$를 선택할 때의 보상 확률분포
- **기댓값**: 해당 슬롯머신을 선택하면 평균적으로 얻는 코인 수

> [!example] 예시
> 슬롯머신 $a$를 선택하여 얻는 코인 개수의 기댓값 = $Q(a)$

---

## 행동 가치 추정

실제로는 진짜 $Q(a)$를 모르기 때문에 **경험을 통해 추정**한다.

### 표본 평균으로 추정

$$\hat{Q}(a) = \frac{\text{지금까지 } a \text{를 선택하여 얻은 보상의 합}}{\text{지금까지 } a \text{를 선택한 횟수}}$$

> [!example] 계산 예시
> **두 대의 슬롯머신으로 한 번씩 플레이:**
> - $Q(a) = 0$, $Q(b) = 1$
>
> **세 번씩 플레이한 결과:**
> $$Q(a) = \frac{0 + 1 + 5}{3} = 2$$
> $$Q(b) = \frac{1 + 0 + 0}{3} = 0.333$$

### 점진적 갱신 방정식 (Incremental Equation)

매번 평균을 새로 계산하는 대신 **점진적으로 업데이트**할 수 있다.

$$Q_{n+1} = Q_n + \frac{1}{n}(R_n - Q_n)$$

여기서:
- $R_n$: $n$번째 행동에서의 보상
- $Q_n$: $n$번째 행동에서의 행동 가치 추정치
- $\frac{1}{n}$: **학습률 (Learning Rate)**

> [!info] 점진적 갱신의 일반형
> $$\text{새 추정치} \leftarrow \text{이전 추정치} + \alpha [\text{목표} - \text{이전 추정치}]$$
> 여기서 $\alpha = \frac{1}{n}$ 이 학습률이다.

## Decision Making 의사결정 — 활용 vs 탐색

강화학습에서 가장 중요한 트레이드오프!

### 1. 활용 (Exploitation) — 지금 아는 최선을 선택
- 현재까지의 결과를 바탕으로 **가장 좋아 보이는 슬롯머신** 선택
- **탐욕적 선택 (Greedy Selection)**
- 단점: 아직 탐색하지 않은 더 좋은 선택지를 놓칠 수 있음

### 2. 탐색 (Exploration) — 새로운 것을 시도
- 어떤 행동이 가치 있는지 정확히 파악하기 위해 **다양한 슬롯머신** 시도
- **무작위 선택 (Random Selection)**
- 단점: 이미 좋다는 것을 아는 선택을 포기하는 비용 발생

> [!warning] 탐색-활용 딜레마
> 탐색만 하면 → 얻은 지식을 활용하지 못함
> 활용만 하면 → 더 좋은 행동을 발견하지 못함
> **둘 사이의 균형이 핵심!**

## ε-탐욕 정책 (ε-greedy)

활용과 탐색을 **확률적으로 조합**하는 가장 기본적인 정책.

$$\text{행동 선택} = \begin{cases} \text{무작위 선택 (탐색)} & \text{확률 } \varepsilon \\ \text{탐욕 선택 (활용)} & \text{확률 } 1 - \varepsilon \end{cases}$$

```
          ε (탐색 확률)
start ──────────────► Arm #1, #2, ... #N (무작위)
      │
      │ 1-ε (활용 확률)
      └────────────► Best Arm (탐욕)
```

| $\varepsilon$ 값 | 동작 |
|-----------------|------|
| $\varepsilon = 0$ | 완전한 활용 (Greedy) |
| $\varepsilon = 1$ | 완전한 탐색 (Random) |
| $0 < \varepsilon < 1$ | **탐색-활용 균형** ← 실전에서 사용 |

> [!tip] 실전 팁
> 초반에는 $\varepsilon$을 크게 설정해 탐색을 많이 하고, 학습이 진행될수록 줄여가는 **ε 감소 전략**이 효과적이다.

## 비정상 밴딧 문제 (Non-stationary Bandit Problem)

### 정상(Stationary) vs 비정상(Non-stationary)

| 구분 | 정상 (Stationary) | 비정상 (Non-stationary) |
|------|-----------------|----------------------|
| 슬롯머신 승률 | 초기 고정값 유지 | **매번 변경됨** |
| 보상 가중치 | 모든 보상에 **동일한 가중치** | 최근 보상에 **높은 가중치** |
| 방식 | 표본 평균 ($\alpha = \frac{1}{n}$) | **고정 학습률** ($\alpha = $ 상수) |

### 고정 학습률 갱신 방정식

$$Q_{n+1} = Q_n + \alpha(R_n - Q_n), \quad \alpha \in (0, 1]$$

이를 전개하면:
$$Q_{n+1} = (1-\alpha)^n Q_1 + \sum_{k=1}^{n} \alpha(1-\alpha)^{n-k} R_k$$

> [!note] 지수 가중 이동 평균
> 고정 학습률을 사용하면 **최근 보상일수록 더 큰 가중치**를 받는다. 이를 지수 가중 이동 평균(Exponentially Weighted Moving Average)이라고 한다.

## 구현 실습

### 실습 #1: `bandit.py` — 기본 밴딧

```python
import numpy as np

class SlotMachine:
    """슬롯머신 (환경)"""
    def __init__(self, q_star):
        self.q_star = q_star  # 진짜 행동 가치

    def play(self):
        # 보상은 정규분포에서 샘플링
        return np.random.randn() + self.q_star

class Agent:
    """에이전트 (플레이어)"""
    def __init__(self, epsilon, action_size):
        self.epsilon = epsilon
        self.Qs = np.zeros(action_size)   # 행동 가치 추정치
        self.Ns = np.zeros(action_size)   # 각 행동 선택 횟수

    def get_action(self):
        if np.random.rand() < self.epsilon:
            return np.random.randint(len(self.Qs))  # 탐색
        return np.argmax(self.Qs)                    # 활용

    def update(self, action, reward):
        self.Ns[action] += 1
        self.Qs[action] += (reward - self.Qs[action]) / self.Ns[action]
```

### 실습 #3: `non_stationary.py` — 비정상 밴딧

```python
class NonStatBandit:
    """비정상 밴딧 환경"""
    def __init__(self, arms=10):
        self.arms = arms
        self.rates = np.random.rand(arms)  # 초기 승률

    def play(self, arm):
        rate = self.rates[arm]
        self.rates += 0.1 * np.random.randn(self.arms)  # 승률 변동!
        return 1 if rate > np.random.rand() else 0

class AlphaAgent:
    """고정 학습률 에이전트"""
    def __init__(self, epsilon, alpha, actions=10):
        self.epsilon = epsilon
        self.alpha = alpha  # 고정 학습률
        self.Qs = np.zeros(actions)

    def update(self, action, reward):
        self.Qs[action] += self.alpha * (reward - self.Qs[action])
```
