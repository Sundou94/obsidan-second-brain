## 1. Bellman Equation – State Value

> **p.19**

### State-Value 함수 유도

State-Value 함수 $v_\pi(s)$는 정책 $\pi$ 하에서 상태 $s$에서 시작했을 때의 **기대 누적 수익(expected return)**입니다.

$$v_\pi(s) \equiv \mathbb{E}_\pi[G_t \mid S_t = s]$$



해당수식을 하나하나 정말 쉽게 풀어서 설명해봐 !! 한국어로 $$v_\pi(s) \equiv \mathbb{E}_\pi[G_t \mid S_t = s]$$

마르코프 결정 과정(Markov Decision Process, MDP)은 의사결정 문제를 모델링하는 수학적 프레임워크입니다. 여기서 우리는 특정 상태에서의 가치(value)를 계산하는 방법을 이해하고자 합니다.

주어진 수식은 다음과 같습니다:

$$v_\pi(s) \equiv \mathbb{E}_\pi[G_t \mid S_t = s]$$

이 수식을 쉽게 풀어 설명해 보겠습니다:

1. **상태 가치 함수(policy value function)**: 
   - $v_\pi(s)$는 특정 정책(policy) $\pi$를 따를 때 상태 $s$의 가치를 나타냅니다.
   - 즉, 이 함수는 상태 $s$에서 시작하여 정책 $\pi$에 따라 행동할 때 기대되는 장기적인 보상(reward)의 합을 의미합니다.

2. **기대값(expectation):**
   - $\mathbb{E}_\pi[\cdot]$는 "기대값"을 나타내며, 특정 확률 분포에 따라 어떤 결과가 나올지 평균적으로 예측하는 것을 의미합니다.
   - 여기서는 주어진 정책 $\pi$에 따라 행동할 때 얻을 수 있는 보상의 평균적인 기대치를 계산합니다.

3. **누적 보상(return):**
   - $G_t$는 시간 $t$부터 시작하여 미래에 받을 모든 보상의 합(즉, 누적 보상)을 의미합니다.
   - 이는 일반적으로 할인 인자(discount factor)를 사용하여 미래의 보상을 현재 가치로 환산한 값의 합으로 표현됩니다.

4. **조건부(expected value given state):**
   - $\mid S_t = s$는 "조건부"를 나타내며, 시간 $t$에 상태가 $s$일 때라는 조건 하에서의 기대값을 계산한다는 것을 의미합니다.
   - 즉, 현재 상태가 $s$일 때 앞으로 얻게 될 누적 보상의 기대값을 구하는 것입니다.

정리하자면, 이 식은 "현재 상태가 $s$일 때 정책 $\pi$를 따르면서 미래에 받게 될 예상 총 보상의 평균값"을 계산한다고 할 수 있습니다. 이는 MDP에서 매우 중요한 개념으로, 각 상태에서 어떤 행동이 최선인지 결정하는 데 사용됩니다.



이를 전개하면:

$$= \mathbb{E}_\pi[R_t \mid S_t = s] + \gamma \mathbb{E}_\pi[G_{t+1} \mid S_t = s]$$

$$= \sum_{a,s'} \pi(a|s)\, p(s'|s,a)\, r(s,a,s') + \gamma \sum_{a,s'} \pi(a|s)\, p(s'|s,a)\, \mathbb{E}_\pi[G_{t+1} \mid S_{t+1} = s']$$

$$\underbrace{\mathbb{E}_\pi[G_{t+1} \mid S_{t+1} = s']}_{= v_\pi(s')} \text{ 로 치환하면}$$

### ✅ Bellman Equation (State Value)


==Bellman 방정식은 마르코프 결정 과정(MDP)에서 최적의 정책을 찾기 위한 핵심 도구입니다==. 이 방정식은 ==특정 상태에서의 장기적인 가치를 계산==하는 방법을 제공합니다. 간단히 말해, Bellman 방정식은 ==현재 상태에서 얻을 수 있는 즉각적인 보상==과 ==미래 상태들에서 기대할 수 있는 최대 가치의 합==으로 이루어져 있습니다.

이 방정식은 다음과 같이 표현됩니다:

\[ V(s) = \max_a \left( R(s, a) + \gamma \sum_{s'} P(s' | s, a) V(s') \right) \]

여기서:
- \( V(s) \)는 상태 \( s \)에서의 가치입니다.
- \( a \)는 가능한 행동(action)을 의미합니다.
- \( R(s, a) \)는 상태 \( s \)에서 행동 \( a \)를 취했을 때 받을 수 있는 즉각적인 보상입니다.
- \( \gamma \)는 할인 요인(discount factor)을 의미하며, 미래 보상의 현재 가치에 대한 중요도를 조절합니다.
- \( P(s' | s, a) \)는 상태 \( s \), 행동 \( a \)를 택했을 때 다음 상태가 \( s' \)일 확률입니다.

==Bellman 방정식은 재귀적인 성격==을 가지고 있으며, 이를 반복적으로 적용함으로써 각 상태의 가치를 업데이트하고 최적 정책을 도출할 수 있습니다. ==이 과정은 동적 계획법(dynamic programming)의 기초==로 사용됩니다.

$$\boxed{v_\pi(s) = \sum_{a,s'} \pi(a|s)\, p(s'|s,a) \left[ r(s,a,s') + \gamma\, v_\pi(s') \right]}$$

**핵심 포인트:**

- **재귀적 관계(Recursive relationship)**: 현재 상태 $s$의 가치와 다음 상태 $s'$의 가치 사이의 관계
- $v_\pi(s_1), v_\pi(s_2), \ldots, v_\pi(s_n)$에 대한 **1차 연립방정식(system of linear equations)** 문제
- Deterministic & Stochastic 모두 적용 가능

$$\boxed{v_\pi(s) = \sum_{a,s'} \pi(a|s)\, p(s'|s,a) \left[ r(s,a,s') + \gamma\, v_\pi(s') \right]}$$
#### 저 수식에 대해서 좀 설명해봐 진짜 진짜 쉽게 설명해봐 
#설명


물론입니다! 이 수식은 마르코프 결정 과정(MDP)에서 정책 \(\pi\)를 따를 때 특정 상태 \(s\)에서의 가치(value)를 계산하는 방법을 설명합니다. 쉽게 말해, 어떤 상태에서 시작했을 때 기대할 수 있는 보상의 총합을 계산하는 것이죠. 이를 좀 더 쉽게 하나씩 풀어볼게요:

1. **\(v_\pi(s)\)**: 이 부분은 상태 \(s\)의 가치입니다. 즉, 상태 \(s\)에 있을 때 정책 \(\pi\)를 따라 행동했을 때 기대할 수 있는 총 보상입니다.

2. **\(\sum_{a,s'}\)**: ==이 기호는 모든 가능한 행동 \(a\)와 다음 상태 \(s'\)에 대해 합계를 구한다는 뜻==입니다. 즉, 모든 가능성을 다 고려한다는 의미입니다.

3. **\(\pi(a|s)\)**: 이는 특정 상태 \(s\)에서 행동 \(a\)를 선택할 확률을 말합니다. 정책 \(\pi\)가 주어졌을 때, 각 행동이 선택될 가능성을 나타냅니다. 폴리시

4. **\(p(s'|s,a)\)**: 이것은 현재 상태 \(s\)에서 행동 \(a\)를 했을 때 다음 상태가 \(s'\)로 변할 확률입니다. 

5. **\[ r(s,a,s') + \gamma\, v_\pi(s') \]**: 이 부분이 핵심인데 두 가지로 나뉩니다:
   - ==**\(r(s,a,s')\)**: 현재 상태에서 행동한 결과로 얻는 즉시 보상입니다.==
   - **\(\gamma\, v_\pi(s')\)**: \(\gamma\)는 할인율이라고 불리며, 미래의 보상을 얼마나 중요하게 생각하는지를 나타내는 숫자(0과 1 사이의 값)이죠. 곱해지는 \(v_\pi(s')\)는 다음 상태 \(s'\)의 기대 가치를 의미합니다.

전체적으로 이 식은 "==현재 상태에서 어떤 행동들을 할지 정하고 그 결과로 얻을 보상과 이후에 도착할 새로운 상태들의 가치를 모두 고려해서 지금 상태의 가치를 구한다=="라는 뜻입니다.

결론적으로, 이 식은 우리가 현재 어디에 있고 앞으로 어떻게 행동할지에 따라 얼마나 좋은 결과를 얻게 될지를 계산하는 것이라 할 수 있습니다!

## 2. 예제: 2-Grid World

> **p.20–24**

### 환경 설정 (p.20)

| 항목 | 설정 |
|------|------|
| 상태 공간 | $S = \{L_1, L_2\}$ |
| 정책 | Stochastic policy (확률적 정책) |
| 태스크 유형 | Continuous task (연속 태스크) |

**Policy $\pi$:**

| Action ($a$) | $s = L_1$ | $s = L_2$ |
|---|---|---|
| left | 0.5 | 0.5 |
| right | 0.5 | 0.5 |

**할인율**: $\gamma = 0.9$

---

### Non-Bellman 방식의 한계 (p.21)

Bellman Equation 없이 무한 급수(infinite series)로 직접 계산하면:

$$v_{\pi_1}(L_1) = 0.5(-1 + 0.9 \times (0.5(-1 + \cdots) + 0.5(1 + 0.9 \times (0.5(0 + \cdots)))))$$

→ **계산 불가 (infinite-series computation problem)** — 풀기 매우 어려움

---

### Bellman Equation 적용 방법 (p.22–23)

**State transition probability** $p(s'|s,a)$ — Deterministic transition:

| 조건 | 확률 |
|------|------|
| $p(s'=L_2 \mid s=L_1, a=\text{RIGHT}) = 1$ | → $L_1$에서 RIGHT → $L_2$ 도달 |
| $p(s'=L_1 \mid s=L_2, a=\text{RIGHT}) = 0$ | |

**연립방정식 도출** ($\gamma = 0.9$):

$$v_\pi(L_1) = \sum_{a \in \{\text{L,R}\}} \pi(a|L_1) \sum_{s'} p(s'|L_1, a)[r + \gamma v_\pi(s')]$$

$$v_\pi(L_2) = \sum_{a \in \{\text{L,R}\}} \pi(a|L_2) \sum_{s'} p(s'|L_2, a)[r + \gamma v_\pi(s')]$$

→ 두 미지수 $v_\pi(L_1)$, $v_\pi(L_2)$에 대한 **1차 연립방정식**을 풀어 각 State Value 계산

## 3. Bellman Equation – Action Value

> **p.25–26**

### Action Value (행동 가치) 정의

$$q_\pi(s,a) = \mathbb{E}_\pi[G_t \mid S_t = s, A_t = a]$$

→ **policy** $= \pi$, **state** $= s$, **action** $= a$일 때 **기대 수익**
→ **Q Function** 이라고도 부름

### State Value와의 관계 (★)

$$v_\pi(s) = \mathbb{E}_\pi[G_t \mid S_t = s] = \sum_a \pi(a|s)\, \mathbb{E}_\pi[G_t \mid S_t = s, A_t = a]$$

$$\boxed{v_\pi(s) = \sum_a \pi(a|s)\, q_\pi(s,a)} \quad \text{(★)}$$

→ State value = 각 action에 대한 action value의 가중 평균 (가중치 = 정책 확률)

### Bellman Equation (Action Value)

$S_t = s$, $A_t = a$에서의 기대수익 $q_\pi(s,a)$:

$$\boxed{q_\pi(s,a) = \sum_{s'} p(s'|s,a) \left[ r(s,a,s') + \gamma \sum_{a'} \pi(a'|s')\, q_\pi(s',a') \right]} \quad \text{(★★)}$$

→ $(s,a)$ 쌍에서의 기대수익이 다음 상태 $(s', a')$의 기대수익과 재귀적 관계

## 4. Optimal Value Function & Policy


#### Optimal Value Function & Policy 에 대해서 짱짱쉬운 설명
#설명 

마르코프 결정 과정(MDP)은 의사 결정 문제를 다루기 위한 수학적 틀입니다. 이를 통해 에이전트가 환경과 상호작용하며 최적의 행동을 선택하는 방법을 찾을 수 있습니다. MDP는 상태, 행동, 보상, 상태 전이 확률로 구성됩니다.

### Optimal Value Function

1. **가치 함수(Value Function)**:  
   ==가치 함수는 특정 상태에서 시작하여 앞으로 받을 것으로 기대되는 보상의 합==을 나타냅니다. 주==어진 정책(policy)을 따를 때 얻을 수 있는 장기적인 보상을 평가==하는 데 사용됩니다.

2. ==**최적 가치 함수(Optimal Value Function)**:==  
   최적 가치 함수는 모든 가능한 ==정책 중에서 가장 큰 기대 보상을 제공하는 정책을 따를 때의 가치 함수==를 말합니다. 즉, 각 상태에서 최선의 행동을 선택했을 때 받을 수 있는 ==최대 기대 보상의 값==을 나타냅니다.
   
   - **상태 가치 함수 \(V^*(s)\)**: 특정 상태 \(s\)에서 시작하여 최적의 정책을 따랐을 때 얻을 수 있는 ==최대 기대 보상==을 나타냅니다.
   
   - **행동 가치 함수 \(Q^*(s, a)\)**: 특정 상태 \(s\)에서 특정 행동 \(a\)를 했을 때 이후 최적의 정책을 따랐을 경우 얻을 수 있는 최대 기대 보상을 나타냅니다.

### Policy

1. **정책(Policy)**:  
   정책은 에이전트가 각 상태에서 어떤 행동(action)을 취할지를 결정하는 규칙이나 전략입니다. 

2. **최적 정책(Optimal Policy \(\pi^*\))**:  
   최적 정책은 모든 상태에서 최대한의 기대 보상을 얻도록 하는 행동 선택 규칙입니다. 최적 정책은 최적 가치 함수를 기반으로 도출됩니다.

### 쉽게 설명하자면:

- ==**Optimal Value Function**은 "각 상황에서 어떻게 행동해야 가장 많은 이익(보상)을 얻을 수 있을까?"라는 질문에 대한 답변==입니다. 마치 게임이나 일상생활에서 '현재 상황'에 따라 '어떤 결정을 내려야 가장 좋은 결과를 얻는지' 계산하는 것과 비슷합니다.

- **Policy**는 "어떤 상황에서는 이렇게 행동하고, 또 다른 상황에서는 저렇게 행동하라"라는 일련의 지침서 같은 것입니다. 예를 들어, 길 찾기 게임에서는 '왼쪽으로 가라', '위쪽으로 가라' 같은 일관된 방향 지시가 될 수 있습니다.

이 두 개념은 MDP 문제를 해결하기 위해 상호작용하며, 궁극적으로 에이전트가 환경 내에서 최고의 성과를 내도록 도와줍니다.
> **p.27**

### Optimal State-Value Function

모든 정책 중 **최대 state-value**:

$$v_*(s) = \max_\pi v_\pi(s)$$

### Optimal Action-Value Function

모든 정책 중 **최대 action-value**:

$$q_*(s,a) = \max_\pi q_\pi(s,a)$$

### Optimal Action $\mu_*$

$$\mu_*(s) = \arg\max_a\, q_*(s,a)$$

### Optimal Policy $\pi_*$

$$\pi_*(a|s) = \begin{cases} 1 & \text{if } a = \mu_*(s) \\ 0 & \text{otherwise} \end{cases}$$

→ Optimal policy는 **deterministic(결정론적)**: 최적 행동에만 확률 1

## 5. Bellman Optimality Equation

> **p.28–29**

### Optimal State-Value (p.28)

일반 Bellman Equation:

$$v_\pi(s) = \sum_{a,s'} \pi(a|s)\, p(s'|s,a) [r(s,a,s') + \gamma v_\pi(s')]$$

**최적화** — $v_\pi(s)$를 최대화하는 $\pi(a|s)$ 선택:

$$\boxed{v_*(s) = \max_a \sum_{s'} p(s'|s,a) [r(s,a,s') + \gamma v_*(s')]}$$

**증명 개요**:
- $\pi(a_1|s) = 0,\ \pi(a_2|s) = 0,\ \pi(a_3|s) = 1$ 형태 (deterministic)
- $v_*(s) = \max\{-2,\ 0,\ 4\} = 4$인 경우, $a_3$에 확률 1 배정

### Optimal Action-Value (p.29)

$$\boxed{q_*(s,a) = \sum_{s'} p(s'|s,a) \left[ r(s,a,s') + \gamma \max_{a'} q_*(s',a') \right]}$$

### Optimal Policy 도출

$$\pi_*(a|s) = \begin{cases} 1 & \text{if } a = \arg\max_{a'} q_*(s,a') \\ 0 & \text{otherwise} \end{cases}$$

> **핵심**: Bellman Optimality Equation은 **최적화 문제(optimization problem)**
> → 비선형(non-linear)이므로 단순 연립방정식보다 어려움
> → **Dynamic Programming** 기법으로 해결

## 6. 예제 풀이

### Example 1: 2-Grid World (p.30–31)

**Bellman Optimality Equation 적용** ($\gamma = 0.9$):

$$v_*(L_2) = \max_a \sum_{s'} p(s'|L_2, a)[r + \gamma v_*(s')]$$

**Optimal Action Value 계산** (★★ 공식):

$$q_\pi(s,a) = \sum_{s'} p(s'|s,a)\{r(s,a,s') + \gamma v_*(s')\}$$
#### 짱짱 쉬운 설명
#설명 

마르코프 결정 과정(MDP)은 의사결정 문제를 수학적으로 모델링하는 방법 중 하나입니다. 여기에서 중요한 개념은 상태(state), 행동(action), 보상(reward), 그리고 상태 전이 확률(transition probability)입니다. 이 모든 요소들이 결합되어 최적의 정책(policy)을 찾는 것이 목표입니다.

**벨만 최적 방정식(Bellman Optimality Equation)**은 MDP에서 최적의 정책을 찾기 위한 기본 도구입니다. 주어진 상태에서 최선의 가치를 계산하는 방법을 제공하죠.

### 1. 벨만 최적 방정식 적용

$$v_*(L_2) = \max_a \sum_{s'} p(s'|L_2, a)[r + \gamma v_*(s')]$$

- **$v_*(L_2)$**: 상태 $L_2$에서의 최적 가치(optimal value)를 의미합니다.
- **$\max_a$**: 각 행동(action) $a$에 대해 가능한 모든 결과를 고려하여 최고의 결과를 선택합니다.
- **$\sum_{s'}$**: 다음 상태 $s'$로 전이될 확률과 그에 따른 가치를 모두 더합니다.
- **$p(s'|L_2, a)$**: 현재 상태가 $L_2$이고 행동 $a$를 취했을 때 다음 상태가 $s'$로 전이될 확률입니다.
- **$r + \gamma v_*(s')$**: 즉시 보상 $r$과 다음 상태의 할인된 가치(discounted value)를 더합니다. 여기서 $\gamma = 0.9$는 할인율(discount factor)을 나타내며, 미래의 보상을 얼마나 중요하게 여길지를 결정합니다.

### 2. 최적 행동 가치 계산

$$q_\pi(s,a) = \sum_{s'} p(s'|s,a)\{r(s,a,s') + \gamma v_*(s')\}$$

- **$q_\pi(s,a)$**: 주어진 정책 $\pi$ 하에서 특정 상태 $s$에서 특정 행동 $a$를 취했을 때의 기대 가치(expected value)입니다.
- **$\sum_{s'} p(s'|s,a)$**: 특정 상태에서 특정 행동을 했을 때 각 가능한 다음 상태 $s'$로 전이될 확률들을 모두 합산합니다.
- **$\{r(s,a,s') + \gamma v_*(s')\}$**: 각 전이 확률에 대해 해당 전이가 주는 즉시 보상과 그 후속 가치를 더한 것입니다.

쉽게 말해, 벨만 방정식은 주어진 정책 하에서 또는 전체적으로 최선의 전략을 찾아내기 위해 현재와 미래 상황을 종합적으로 고려하여 의사결정을 내리도록 도와줍니다. 이 과정은 MDP 문제 해결에 있어 매우 핵심적인 역할을 합니다.



**State $L_1$에서:**

| Action | 계산 | 값 |
|--------|------|-----|
| LEFT | $-1 + 0.9\, v_*(L_1)$ | 3.734 |
| RIGHT | $+1 + 0.9\, v_*(L_2)$ | 5.257 |

$$\therefore \mu_*(L_1) = \text{RIGHT}$$

**State $L_2$에서:**

| Action | 계산 | 값 |
|--------|------|-----|
| LEFT | $0.9\, v_*(L_1)$ | 4.734 |
| RIGHT | $-1 + 0.9\, v_*(L_2)$ | 3.257 |

$$\therefore \mu_*(L_2) = \text{LEFT}$$

## 7. 요약

> **p.35**

| 개념 | 방정식 유형 | 풀이 방법 |
|------|------------|-----------|
| **Value Function** $v_\pi(s)$ | **Bellman Equation** → 연립방정식 문제 | 선형 시스템 풀이 |
| **Optimal Value Function** $v_*(s)$ | **Bellman Optimality Equation** → 최적화 문제 | Dynamic Programming |

### 핵심 수식 정리

| 이름 | 수식 |
|------|------|
| State Value (Bellman) | $v_\pi(s) = \sum_{a,s'} \pi(a|s)\, p(s'|s,a)[r + \gamma v_\pi(s')]$ |
| Action Value (Bellman) | $q_\pi(s,a) = \sum_{s'} p(s'|s,a)[r + \gamma \sum_{a'} \pi(a'|s') q_\pi(s',a')]$ |
| State-Value 관계 | $v_\pi(s) = \sum_a \pi(a|s)\, q_\pi(s,a)$ |
| Optimal State Value | $v_*(s) = \max_a \sum_{s'} p(s'|s,a)[r + \gamma v_*(s')]$ |
| Optimal Action Value | $q_*(s,a) = \sum_{s'} p(s'|s,a)[r + \gamma \max_{a'} q_*(s',a')]$ |
| Optimal Action | $\mu_*(s) = \arg\max_a\, q_*(s,a)$ |

### 전체 흐름

```
MDP 정의
  └─> Bellman Equation (연립방정식) ─> v_π(s), q_π(s,a) 계산 가능
         └─> Bellman Optimality Equation (최적화)
                └─> v_*(s), q_*(s,a) 계산
                       └─> Optimal Policy π* 도출
                              └─> Dynamic Programming으로 효율적 풀이
```

---

*참고: 수식에서 $\pi(a|s)$는 state $s$에서 action $a$를 선택할 확률, $p(s'|s,a)$는 state transition probability, $r(s,a,s')$는 reward, $\gamma$는 discount factor.*
